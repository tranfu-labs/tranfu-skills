# Pressure Test

完成研究综合后，必须切换到反方视角，主动寻找这个需求为什么会失败。

## 一、必问的七类致命问题

1. 用户真的会行动，还是只是口头认可
2. 需求是高频刚需，还是低频抱怨
3. 现有替代方案是否其实已经“足够好”
4. 获客成本是否会高到吃掉价值
5. 团队是否缺少关键资源、渠道或专业门槛
6. 竞品、平台或大厂是否能快速复制
7. 合规、数据、交付或服务链路中是否存在硬性阻断

## 二、压力测试输出方式

至少列出：

- 生死线问题
- 为什么它危险
- 如果判断错误，会造成什么后果
- 怎样低成本验证
- 如果验证失败，应该如何收窄、转向或停止

## 三、反方口吻建议

更合适的风格是：

- “如果这件事成立，最先应该出现什么证据？现在有吗？”
- “这是不是一个听起来痛，但不会驱动购买的场景？”
- “如果用户继续用 Excel、人工流程或现有平台勉强解决，你凭什么切进去？”
- “如果只能服务很小的一群人，这个需求还能撑起什么投入规模？”

## 四、正向压力测试

在完成反方压力测试后，可补做一次正向压力测试，避免因为过度保守错杀方向。

可以额外追问：

- 如果这件事成立，最大的上行空间是什么
- 是否存在被低估的机会或被忽略的优势
- 哪个积极信号一旦出现，会显著提高继续投入的理由

正向压力测试不是为了对冲风险，而是为了识别“值得继续验证”的上行条件。
