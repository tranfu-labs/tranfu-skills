# product-requirement-analysis

对模糊产品想法、需求方向或新功能机会做分阶段需求验证。它帮助用户判断需求是否真实存在、是否值得投入、适合从哪个切入点继续验证。

## 什么时候用它

适合在还没有进入 PRD 或研发实现前使用：用户只有初步想法、新需求、新功能方向，想做市场调研、竞品分析、用户声音挖掘、需求强度判断、风险识别和下一步验证建议。

## 怎么用 (触发示例)

跟 Claude 说:
- "帮我分析这个需求值不值得做"
- "我有个产品方向, 想做一轮需求验证"
- "帮我调研这个功能机会, 看看市场、竞品和用户痛点"
- "快速判断一下这个需求有没有真实用户价值"

## 你会看到什么

运行后会按阶段产出 intake、expanded brief、raw research、综合分析、verification report、research gate、pressure test、final recommendation 等文件。最终会给出是否继续、为什么、主要风险、数据缺口和下一步最值得执行的验证动作。
