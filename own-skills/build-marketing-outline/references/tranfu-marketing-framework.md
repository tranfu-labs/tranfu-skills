## 1. Brand System

Treat `tranfu.com` as a publicly built Agent-native organization experiment, not as a generic AI-tool directory.

Use the long-term narrative `Agent 公司养成记`: real workflow friction becomes a focused tool; the tool tests a scene; content exposes the evidence and learning; the website retains the tool, experiment, method, and feedback.

Every product story should perform five jobs without saying all five explicitly:

1. Bring a user in through a concrete problem.
2. Show a real Agent or vibe-coding capability sample.
3. Build trust through a traceable experiment.
4. Test whether the scene deserves further investment.
5. Reinforce the TranFu brand system.

Use one concise brand connection near the end. The user's problem and result must remain the lead.

## 2. Translation Model

Translate the product through exactly one main chain:

`场景 -> 冲突 -> 实验 -> 结果 -> 回流`

- **场景**: one audience at one concrete moment.
- **冲突**: the single most costly or frustrating step in the old method.
- **实验**: why TranFu turned that friction into a focused tool and what the build tests.
- **结果**: the first observable output or state change, backed by evidence.
- **回流**: how the product connects to tranfu.com, the experiment record, and one user action.

Do not replace this chain with a feature list.

## 3. Tool Roles

Choose one primary role and at most one secondary role.

| Role | Meaning | Marketing consequence |
|---|---|---|
| 引流工具 | Bring new users into TranFu | Emphasize immediate scene, result, and low-friction entry |
| 能力展示工具 | Demonstrate Agent/vibe-coding capability | Emphasize process evidence and reusable capability |
| 场景验证工具 | Test whether a use scene has demand | Make feedback questions and learning signals explicit |
| 赛道探针工具 | Test a possible commercial direction | State the hypothesis without presenting it as validated demand |
| 资源站配套工具 | Support an existing content/resource system | Emphasize contextual utility and internal navigation |
| Agent 化过程副产物 | Externalize an internal efficiency tool | Emphasize the real friction and honest experimental status |

## 5. Hook Selection

Generate two or three candidates from this radar:

| Hook | Test |
|---|---|
| 高频痛点 | Does the primary audience encounter it often? |
| 低门槛结果 | Can the user reach a useful result with little learning? |
| 前后对比 | Is the state change immediately visible or measurable? |
| 视觉冲击 | Can one image or short recording explain the value? |
| 情绪共鸣 | Will the audience recognize the moment as their own? |
| 方法可复制 | Does the experiment expose a reusable method or SOP? |
| Agent 感 | Does it show meaningful work delegated to an Agent flow? |
| 赛道信号 | Can usage or feedback test a possible direction? |

Score each candidate from 0-2 on:

- **广度**: frequency within the primary audience.
- **直观度**: ability to explain with one image or a 15-second recording.
- **匹配度**: fit with the primary audience and selected launch channel.

Choose the highest total. On ties, prefer `前后对比`, then `低门槛结果`, then `视觉冲击`. Record rejected hooks as follow-up topics. Use only the winner in the main launch outline.

## 6. Content Archetypes

Choose one primary archetype.

| Archetype | Use when | Core structure |
|---|---|---|
| 结果展示型 | The result is visible or comparable | Result -> scene -> short path -> experiment -> return |
| 问题解决型 | One audience has one clear friction | user moment -> old friction -> focused solution -> result -> return |
| 实验过程型 | The build process demonstrates Agent practice | problem origin -> tool decision -> process -> result -> asset -> return |
| 方法沉淀型 | The tool exposes a reusable method | tool as surface -> method -> reusable steps -> Agent opportunity -> return |
| 机会验证型 | The product tests a possible market direction | scene hypothesis -> experiment -> observed signal -> next test -> return |

## 7. Channel Responsibilities

Do not duplicate one outline across channels.

| Channel | Primary job | Editorial shape |
|---|---|---|
| 小红书 | Make the target user want to try or save | Result-led 6-8 image/short-video beats; concrete scene and visual proof |
| 微信公众号 | Build trust and retain method | 6-8 section experiment or method review with evidence, limits, and learning |
| 知乎 | Answer a real searchable question credibly | Conclusion first, then problem, method, evidence, tradeoffs, reusable insight |
| 头条号 | Explain utility to a broad audience | Plain-language scene, old method, direct benefit, short process, limits |
| 微博 | Create launch momentum and interaction | A short launch sequence: fact, result, and feedback/iteration posts |
| 官网 | Convert and retain trust | Tool page for use; experiment page for why, evidence, limits, and next step |

## 8. Launch Sequence And Return Path

Use this default D0-D7 sequence and adjust only when product evidence justifies it:

| Time | Action |
|---|---|
| D0 | Website tool/experiment pages and Weibo launch fact |
| D1 | Xiaohongshu result-led launch |
| D2 | Toutiao plain-language explanation |
| D3 | WeChat deep experiment review |
| D4 | Zhihu problem answer |
| D5 | Secondary angle from a rejected hook |
| D7 | Evidence update, feedback review, and next experiment note |

Every channel outline must reserve the return trio:

1. One audience-appropriate TranFu brand connection.
2. One traceable entry using `{URL}?utm_source={channel}&utm_medium=social&utm_campaign={experiment_id}` when a public URL exists.
3. One primary action; allow a second action only when it does not compete with the first.

Use source codes `xhs`, `wx`, `zhihu`, `toutiao`, `weibo`, and `site`.

## 9. Editorial Rules

- Lead with a solved problem or observable result, not the company or technology.
- Keep one main audience, scene, result, hook, and CTA across the launch matrix.
- Show results before features.
- Preserve the truth that this is a focused product experiment; do not imitate mature-product certainty.
- Do not invent feedback, testimonials, accuracy, time savings, price, free access, privacy guarantees, commercial rights, or market demand.
- Separate page claims, source implementation, existing results, and end-to-end tests.
- State meaningful limits where a user could otherwise make a wrong decision.
- Keep the brand connection concise and practical.
