# 六大思维原则 · 深度操作指南

> SKILL.md 中的六大原则是顶尖研究员心智的核心。本文档提供每条原则的详细操作方法、正反例对照，以及常见陷阱。AI 在执行复杂研究任务时应参考此文档。

---

## 原则 1：第一性原理优先于框架套用

### 核心动作

任何研究任务开始前，先用三个根本问题"打地基"：

1. **谁在为什么样的痛点付什么样的价格？**（交易结构）
2. **这个交易结构在什么条件下会改变？**（变化机制）
3. **我能在这个结构中占据什么位置？**（机会定位）

### 反例 vs 正例

❌ **框架先行（平庸）**：
> "我们用 PESTEL 分析这个市场。政治方面…经济方面…"

✅ **第一性原理（顶尖）**：
> "中小企业财税 SaaS 这个市场，本质上的交易结构是：会计师事务所/代账公司用人力替中小企业处理高频低价值的合规工作（200-500元/月）。这个结构的脆弱点在于：1）人力成本逐年上升 2）AI 已能完成 80% 的标准化工作 3）但客户决策方（老板）不直接使用产品，使用方（财务）有抵触。所以真正的机会窗口不是替代代账公司，而是…"

### 常见陷阱

- **陷阱 A**：把"做了 PESTEL 分析"当成"分析过了"。框架是检查清单，不是答案。
- **陷阱 B**：用框架掩盖思考的缺失。如果一个研究全是"政治、经济、社会、技术"的罗列，没有交易结构的洞察，它就是失败的。

## 原则 2：用数据说话，拒绝形容词分析

### 操作标准

每个关键判断必须包含四要素：**具体数字 + 来源 + 时间 + 推理逻辑**。

### 形容词黑名单

以下表达在报告中**禁止出现**（除非紧跟具体数据）：

- "潜力巨大""前景广阔""空间广阔"
- "增长迅速""快速发展""蓬勃发展"
- "竞争激烈""玩家众多""百花齐放"
- "需求旺盛""痛点强烈""刚需明显"
- "技术领先""模式创新""生态完善"

### 替换模板

| ❌ 形容词版 | ✅ 数据版 |
|------------|----------|
| "市场潜力巨大" | "2024 年 324 亿元，2027 年预计 612 亿元（艾瑞 2025-03），CAGR 23.5%" |
| "竞争激烈" | "Top 5 玩家市占率合计 38%，平均毛利率从 2022 年的 62% 降至 2024 年的 41%（SensorTower）" |
| "需求旺盛" | "知乎相关话题 2024 年讨论量同比增长 340%，TOP 100 帖子平均 1.2K 评论" |
| "技术领先" | "推理速度比第二名快 2.3 倍（MMLU benchmark），但成本高 1.8 倍" |

### 例外情况

执行摘要中允许使用一定的形容词以提升可读性，**但必须在正文用数字佐证**。如果正文也找不到数字，那个形容词就不能用。

## 原则 3：三层标注法

### 三层定义

| 层级 | 标记 | 标准 | 示例 |
|------|------|------|------|
| 事实 | 【事实】 | 公开数据/可验证证据 | 【事实】OpenAI 2024 年营收 37 亿美元（The Information 2024-12） |
| 推断 | 【推断】 | 基于事实的逻辑外推（须说明推理链） | 【推断】按当前 30%/月的增速外推，OpenAI 2025 年底 ARR 有望突破 100 亿美元（推理：12 月基数 × 1.3^N） |
| 观点 | 【观点】 | 主观判断（须说明依据 + 置信度） | 【观点】OpenAI 的护城河比 Google 想象的弱（置信度：中。依据：Anthropic/xAI 模型差距已缩小到 6 个月内，企业客户多采用多模型策略） |

### 为什么必须区分

平庸研究最大的特征是**把观点伪装成事实**。例如：

- "中国 AI 应用市场将在 2025 年迎来爆发" — 这是观点，不是事实
- "用户更愿意为 Agent 付费" — 这是推断，需要数据支撑

读者必须知道每个判断的"硬度"，才能判断风险。

### 操作建议

不需要每句都标注，但**每个章节的核心结论**必须明确层级。在执行摘要、机会洞察、反共识章节，标注密度应更高。

## 原则 4：寻找拐点而非描述现状

### 拐点的定义

拐点 = **趋势加速度发生变化的点**。不是"现在的趋势是什么"，而是"什么正在让现在的趋势加速/减速/反转"。

### 拐点的五种来源

1. **政策拐点**：监管松绑/收紧、新补贴、新试点（如 2024 年具身智能政策）
2. **技术拐点**：成本曲线断崖式下降、能力解锁新场景（如 GPT-4o 多模态）
3. **需求拐点**：人口结构变化、代际偏好迁移、宏观周期切换
4. **供给拐点**：关键资源解锁/枯竭、产能突变、竞争格局重组
5. **资本拐点**：钱的流向反转、估值体系重构、退出渠道变化

### 拐点输出模板

每个拐点必须包含：

```
拐点名称：[简短描述]
信号类型：[政策/技术/需求/供给/资本]
触发证据：[具体事件 + 时间 + 来源]
方向判断：[上升 / 平稳 / 下行]
信号强度：[强 / 中 / 弱]
影响传导：[对哪些细分赛道有什么具体影响]
时间窗口：[预计 N 个月内显现]
```

### 反例

❌ **现状描述（平庸）**：
> "AI Agent 市场正在快速发展，越来越多公司推出 Agent 产品。"

✅ **拐点识别（顶尖）**：
> 拐点：MCP 协议标准化（2025 H1）。
> 触发证据：Anthropic 2024-11 推出 MCP，2025 Q1 OpenAI/Google 跟进。
> 方向判断：上升。强度：强。
> 影响传导：① 单点工具类 Agent 商业模式贬值（互联互通后，集成成本 → 0）② 数据/场景独占者议价权上升 ③ 出现"Agent 中间层"新物种机会窗口约 9-12 个月。

## 原则 5：反共识思维

### 为什么这是顶尖研究员的核心

研究员的价值 = 你的判断 - 市场共识。如果你的报告全是大家都已经知道的事，价值为零。

**真正的 alpha 来自三种情况**：

1. **共识是错的**（最高价值）
2. **共识是对的，但市场低估了程度**
3. **共识对了一半**（看到了 A 没看到 B）

### 反共识三步法

**第一步：识别共识**

问自己：如果我去任何投资人路演会议、行业大会，关于这个市场，所有人都会点头同意的 3 句话是什么？

**第二步：压力测试**

针对每条共识问：
- 这个共识基于什么假设？
- 这个假设在什么条件下会失效？
- 历史上类似的共识有过被推翻吗？为什么？

**第三步：输出反共识判断**

格式：
```
市场共识：[一句话总结]
我的判断：[同意 / 部分同意 / 不同意]
非共识洞察：[具体观点]
支持证据：[2-3 条具体证据]
置信度：[高 / 中 / 低]
反转条件：[什么数据出现会让我改变看法]
```

### 经典反共识案例参考

| 时期 | 共识 | 非共识（事后被证明对的） |
|------|------|--------------------------|
| 2019 | 在线教育是百亿美金赛道 | 监管风险被严重低估 |
| 2021 | 元宇宙是下一代互联网 | 硬件没有就绪，需求被高估 |
| 2022 | LLM 是 OpenAI 一家通吃 | 开源会快速追上 |
| 2023 | AI 应用层会出现快速爆发 | 模型能力不够，应用层得等 |
| 2024 | AGI 还需要 5-10 年 | Reasoning 模型让时间表大幅前移 |

### 陷阱：为反而反

反共识不是为了标新立异，而是基于**更深的事实和逻辑**。如果你的非共识观点没有比共识更强的证据支撑，它就只是噪音。

**自检**：你的非共识判断如果错了，你愿意用什么打赌？如果不愿意打赌，说明你自己也没那么相信。

## 原则 6：脆弱性思维（事前验尸）

### 核心动作

在报告完成之前，强制做一次"事前验尸"（pre-mortem）：

> **想象现在是 2 年后，这份报告的核心判断被证明是错的。请描述：错在哪里？为什么会错？最早的预警信号本应是什么？**

### 输出模板

```
情景：报告核心判断 [X] 被证明错误。

最可能的失败原因 Top 3：
1. [原因] — 为什么会这样：[机制] — 早期信号：[可观测指标]
2. ...
3. ...

我会改变看法的具体触发条件：
- 如果 [指标 A] 出现 [变化]，我将 [调整判断]
- 如果 [事件 B] 发生，整个机会窗口关闭
- 如果 [数据 C] 持续 [N 个月] 没达到 [阈值]，需要重新评估

风险类别评分：
- 数据风险：__/100
- 技术风险：__/100
- 商业风险：__/100
- 监管风险：__/100
- 综合脆弱性：__/100
```

### 为什么这是顶尖标志

平庸分析：只讲机会，不讲怎么会输。

顶尖分析：**先把所有可能输的方式列清楚**，再给出"这些风险都已经被识别和定价后"的净判断。

读者真正信任的研究员，不是那种永远乐观的，而是那种能告诉你"如果我错了，最可能错在哪"的。

### 触发条件的重要性

模糊的"有风险"等于没说。必须给出**可观测、可触发**的具体条件。例如：

- ❌ "如果监管收紧，业务会受影响"
- ✅ "如果 2025 Q3 前网信办发布生成内容标识强制规范，C 端娱乐类应用 LTV 将下降 30%+，需立即调整商业模式"

## 六原则的协同关系

这六个原则不是独立检查项，而是一个**思维系统**：

```
原则 1（第一性原理）→ 提供深度
原则 2（数据为本）→ 提供硬度
原则 3（三层标注）→ 提供透明度
原则 4（寻找拐点）→ 提供前瞻性
原则 5（反共识）→ 提供差异化价值
原则 6（脆弱性）→ 提供可信度
```

**深度 + 硬度 + 透明度 + 前瞻性 + 差异化 + 可信度 = 顶尖研究员的输出特征**

任一原则缺失，都会让研究的质量等级下降一级。
