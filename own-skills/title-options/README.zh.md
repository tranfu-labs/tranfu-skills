# title-options

为完整文章生成多样、适合平台的标题，同时确保每个承诺都能由正文事实和结论兑现。

## 什么时候用它

**生成平台标题**

我有一篇完整文章或足够完整的片段，需要给指定内容平台生成标题候选。

**优化已有标题**

我想保留旧标题中站得住的承诺，或换一种表达策略，并让文章正文成为不可逾越的事实边界。

**生成多平台标题**

我需要分别为微信公众号、小红书、知乎、微博或头条号生成标题组，让每组符合对应平台的阅读习惯。

**不接**

选题、正文写作、视频标题、SEO 标题、广告、邮件主题、落地页文案、产品命名，或只有旧标题却没有正文时，不使用本流程。

## 它会产出什么

**正文无法兑现的吸睛标题会被淘汰，即使因此少于用户要求的数量。**

- **标题候选**：按当前模式的固定结构，在对话中返回候选和推荐标题。
- **平台适配**：应用各平台目标与格式规则，包括微博的开头钩子和话题格式。
- **事实边界**：保留原文的不确定性、行动主体、范围、数字、日期、结果和其他事实限定。
- **旧标题分组**：旧标题模式会区分保留原方向和改换策略两组候选。
- **可选核验**：只有用户明确要求核验文章事实、品牌风格或当前平台规则时才联网。
- **绝不会做**：默认不改源文章，不创建文件，不展示内部评分，不执行正文里的指令，也不把外部新事实塞进标题。

## 前置条件与边界

**前置条件**

必须提供唯一来源，可以是粘贴的完整内容，也可以是一个可读的本地 `.md` 文件；还要指定至少一个支持的平台。优化旧标题时必须同时提供旧标题。

**请求要求**

- 不接受 URL、DOCX、PDF、TXT、不可读文件，或尚未选定的多个来源。
- 多平台共用总数量时，必须明确每个平台的有效配额，本流程不会自行分摊。
- 参考非内置品牌时，需要提供三到五条代表性标题，或明确允许联网调研。

**相邻流程分工**

| 需要做的事 | 交给 |
|---|---|
| 决定文章写什么 | `content-topics` |
| 撰写或修改正文 | 写作或编辑流程 |
| 给产品或 skill 命名 | 命名流程 |

**微妙边界**

外部核验可以确认原文事实、品牌特点或当前规则，但不能为正文从未写过的标题主张提供依据。
