# write-spec
从一个模糊想法或功能需求，生成结构化的 PRD / feature spec。

## 什么时候用它
- 需要把一个模糊想法变成正式 spec
- 要 scope 一个功能，定义目标和非目标
- 需要拆分大需求为分阶段 spec

## 怎么用 (触发示例)
跟 Claude 说:
- "帮我写个 SSO 功能的 spec"
- "把这个想法变成 PRD"
- "scope 一下这个功能"

## 你会看到什么
一份包含 Problem Statement、Goals、Non-Goals、User Stories、Requirements (P0/P1/P2)、Success Metrics、Open Questions、Timeline 的结构化文档。
