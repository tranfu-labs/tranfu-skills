# 微信公众号

## 入口与范围

- 后台：`https://mp.weixin.qq.com/`
- 默认采集用户增长和内容趋势。
- 默认使用截至昨天的 30 个完整自然日。起始日前一天的期初行只在内存中用于对账，不写入 manifest。

## 优先流程

1. 让用户扫码并选择正确公众号。
2. 通过可见菜单进入用户分析/用户增长，选择明确起止日期，只点击一次下载并解析 `user_analysis.xls`。
3. 进入内容分析/全部内容分析，使用同一日期范围，只点击一次下载并解析 `tendency_*.xls`。
4. 验证账号名称、文件签名、日期集合、每日字段和页面汇总。

后台 URL 常含认证查询参数。`source_url` 只能保存 `https://mp.weixin.qq.com/<path>`，不得复制 query 或 fragment。

## 可见数据回退

- 用户增长：使用可见分页，逐行读取时间、新关注、取消关注、净增和累积关注；页码不前进或边界记录重复时停止。
- 内容趋势：依次选择阅读、分享、跳转阅读原文、微信收藏、发表篇数；优先读取可见 tooltip、数据标签或无障碍文本中的原值。
- 不得仅凭 SVG 路径、绘图区高度或像素坐标把估算值标记为 `exact`。没有精确文本时标记 `partial`，并省略相应规范指标。
- 每个精确系列必须覆盖完整日期集合，并与页面或导出总量一致。

## 校验与限制

- 校验 `净增关注人数 = 新关注人数 - 取消关注人数`；期初行只用于验证关注总量连续性。
- 原生下载未被 Codex 接收时，只执行一次可见数据回退，不重复点击下载。
- 图表只提供“全部渠道”时，不声称包含传播渠道或内容来源明细。
- 不操作群发、草稿、留言和账号设置。
