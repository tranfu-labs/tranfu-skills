回头看 agent-harness 这一年走过的路，从最早的一个简单 chatbot 雏形，到现在的多租户 Agent Workflow 平台，团队经历了一段不太平静的日子。这篇文章是对这一年的一次梳理。

## 探索期

在最开始的几个月，我们对自己要做什么产品其实是不清楚的。最早期的版本是一个把 GPT-3.5 当 if-else 用的 chatbot，用户可以来问几个固定场景的问题。那段时间团队规模很小，每天大家就是写 prompt、调 prompt、上线、收反馈，节奏比较快。

我们那时候的核心指标是用户使用量。整体用户数在第一个月增长了 380%，token 使用量也在持续上升。客户对早期版本的反馈整体是正面的，大家普遍觉得这个方向是有想象空间的。

我们也有一些不太顺的时候。比如某一段时间，token 月度账单出现了一个比较夸张的跳跃，大概上涨了 220%，这迫使我们开始考虑成本结构的优化。我们对模型调用做了一轮整理，把一些重复的调用合并掉，整体调用次数下降了 35%。

## 成长期

随着用户基数的扩大，我们开始进入第二个阶段。这个阶段我们做了几个比较重要的事情：

第一，我们重新审视了产品定位。最早我们把自己定位成一个"垂直行业 chatbot"，但随着客户群体的扩展，我们慢慢觉得这个定位太窄了。我们对产品做了一次重新定义，转向 Agent Workflow 平台。这次转型让我们的客户覆盖面提升了 150%。

第二，我们做了 orchestration 层的引入。这个改造让单次会话的平均 token 成本下降了 73%，准确率提升了 41%。这是一次比较成功的工程改造。

第三，我们开始引入更多客户。在这段时间内，付费客户数从某个起点增长了 4 倍，月度调用量增长了接近 6 倍。整体来看，第二阶段是一个相对顺风顺水的阶段，团队对自己做的事情更有信心了。

## 突破期

到了第三阶段，我们开始尝试一些更激进的产品形态。最重要的一个动作是开放 workflow 自定义——我们让付费客户可以在自己的 UI 里直接拼装 workflow。这个动作风险不小，但回头看是值得的。

开放后第一个月，自定义 workflow 的使用率达到了 45%。SLA 整体下降了 12%，但我们认为这是一个可以接受的代价。月度调用量再翻了 3 倍，付费客户数也增长了 60%。

这一阶段我们对团队也做了一些调整。团队规模扩大了 50%，新的同事加入之后，整体的执行节奏更快了。我们对未来一年的规划也变得更清晰。

## 截至 2025-04-30 的当前状态

- 注册客户：相比 2024-08 增长了 12 倍
- DAU：相比 2024-08 增长了 6.6 倍
- 月度 token 账单：相比 2024-08 上升了 256%
- 团队规模：扩大了 200%

## 还没解决的问题

- 多租户的成本归因还在持续优化中。
- eval 平台是自建还是接第三方，我们还在权衡。

## 一个真问号

如果客户量再翻三倍，现在的架构还撑得住吗？我们觉得是有挑战的。
