# 素材调研与核查

在中文 AI 与科技内容动笔前，先建立一套有来源、有结论边界的调研材料。

## 什么时候用它

**调研科技话题**

我准备写 AI 模型、开发工具、芯片、机器人或互联网平台事件，想先查清事实、争议和未知信息，再进入写作。

**补充现有材料**

我手里已有链接、Markdown、文本或 PDF，希望把它们当作线索，与独立来源交叉核实，整理成可以安全使用的主张。

**发现近期选题**

我需要近期且有证据支撑的选题候选，不想只看标题列表。希望每个候选都能追溯到具体来源，再由我选择。

**不接**

它不处理非科技调研、单点事实查询、成稿改写、标题生成或内容发布。

## 它会产出什么

**搜索结果只有打开原始页面并记录后，才会成为证据。**

- **独立运行**：在 `reference/collect-sources/` 追加调研简报、来源笔记和编辑简报
- **总控运行**：向 `content-production` 返回经过校验的调研包，不另建第二套运行记录
- **证据链路**：给来源和主张分配稳定编号，记录冲突，并标明可用、存疑或不可用
- **调研状态**：以完成、部分完成或阻断收尾，明确列出尚未解决的问题
- **外部动作**：搜索并读取公开网页，只写指定的调研文件
- **绝不会做**：代写文章、修改用户原始材料、要求登录，或保存整篇受版权保护的网页

## 前置条件与边界

**前置条件**

需要一个 AI 或相邻科技话题、可写的工作目录，以及公开搜索和网页读取能力。用户提供的本地材料必须可以读取。

**相邻能力分工**

| 需求 | 交给 |
|---|---|
| 跑完整内容交付 | **content-production** |
| 根据证据写稿 | **draft-content** |
| 检查完成后的文章 | **credibility-review** |

**不接的场景**

- 与科技内容生产无关的调研
- 需要登录、付费、验证码或人工验证的页面
- 写作、排版、配图或发布最终内容

**容易混淆的边界**

- 复合写作请求可在收到可用交接材料后继续，但本能力本身停在调研阶段
- 可靠来源互相冲突时会保留双方说法，不会擅自选一边
- 没有可追溯证据时会明确阻断，不会用薄弱话题凑数
