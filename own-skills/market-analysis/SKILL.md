# 全景市场分析技能 v2（AI原生方法论版）

> 传统 PEST/五力/SWOT 在AI时代速度、颗粒度、动态性均不足。本技能采用**AI原生新方法论**，覆盖赛道选择 → 项目诊断 → 调研落地 → 机会验证四大模块，输出可直接落地决策的深度报告。

---

## 执行总流程

```
输入市场关键词
    ↓
Step 1：边界确认（10秒判断）
    ↓
Step 2：12维度并行搜索（核心！）
    ↓
Step 3：信息评分与筛选
    ↓
Step 4：AI原生框架分析（四大模块）
    ↓
Step 5：输出结构化报告 + 机会洞察
```

---

## Step 1：边界确认

收到分析请求后，先用 **10秒** 判断：

| 问题 | 处理方式 |
|------|----------|
| 市场边界太宽（如"AI市场"）？ | 先做全景，报告末尾建议细分方向 |
| 地域范围未说明？ | 默认：**中国市场为主 + 全球对比** |
| 分析深度未指定？ | 默认：**标准深度（12-16次搜索）** |
| 用户说"快速看看"？ | 切换**快速模式**（6-8次搜索） |
| 用户说"深入研究"？ | 切换**深度模式**（20+次搜索） |

## Step 2：12维度并行搜索

按以下维度搜索，**每个维度至少1次**，重要维度用 `web_fetch` 获取完整内容。

### 模块A：赛道洞察（4个维度）

**A1 — 市场规模与增速**（必搜）
```
搜索词："{市场} 市场规模 2024 2025 亿 CAGR"
搜索词："{market} market size revenue forecast 2025 2026"
目标：找到具体数字，不接受"潜力巨大"这类泛描述
```

**A2 — 拐点信号捕捉**（关键！AI原生视角）
```
搜索词："{市场} 政策 2024 2025 试点 补贴 监管"
搜索词："{市场} 技术突破 新进展 2025"
搜索词："{市场} 搜索热度 用户关注 增长"
目标：找"正在发生的拐点"——政策松绑/收紧、技术跃迁、需求爆发
```

**A3 — 精准TAM测算**
```
搜索词："{市场} 用户规模 渗透率 付费用户"
搜索词："{市场} 目标人群 画像 规模 城市"
目标：分城市、分人群、分场景拆解真实可切入市场（SOM），拒绝泛TAM
```

**A4 — 非对称机会信号**
```
搜索词："{市场} 用户抱怨 痛点 吐槽 不满"
搜索词："{市场} 空白 没有 解决不了 site:zhihu.com OR site:reddit.com"
目标：找"高痛点+传统方案成本高/解决不好"的机会窗口
```

### 模块B：竞争格局（3个维度）

**B2 — 量化竞争强度**

搜索词："{市场} 竞品对比 差异化 护城河"
搜索词："{市场} 用户迁移成本 转换成本 粘性"
目标：评估进入壁垒、替代品威胁、上下游议价力
```

**B3 — 竞品深度拆解**
```

目标：从竞品弱点找差异化切入口

## Step 3：信息评分标准

| 维度 | 权重 | 说明 |
|------|------|------|
| 时效性 | 30% | 2025年=满分，2024年=0.8，2023年=0.5，更早递减 |
| 来源可信度 | 30% | 行业报告/官方数据 > 主流媒体 > 公司公告 > 论坛讨论 |
| 数据具体性 | 25% | 有数字/案例 > 泛泛描述（"潜力巨大"之类直接降权） |
| 相关性 | 15% | 直接相关 > 间接参考 |

**优先来源**：艾瑞咨询、36氪、IT桔子、虎嗅、CB Insights、Gartner、Bloomberg、行业协会官网

## Step 4：AI原生框架分析（四大模块）

搜索完成后，按以下框架进行分析：

### 模块一：赛道选择分析

**① AI-PESTEL 拐点分析**
不是罗列现状，重点识别"正在发生的变化信号"：
- 政策：有无新试点/补贴/监管收紧？→ 政策风险/机遇窗口
- 经济：消费意愿变化对该市场的影响
- 技术：近期技术突破是否降低成本或打开新场景？
- **每项必须给出"拐点判断"：上升/平稳/下行趋势**

**② 量化五力评分**（0-100分，越高代表该力越强）
```
新进入者壁垒：__/100  （数据/技术/合规/渠道壁垒）
替代品威胁：  __/100  （AI替代率/用户迁移成本）
供应商议价：  __/100  （集中度/可替代性）
买家议价：    __/100  （集中度/价格敏感度）
行业内竞争：  __/100  （玩家数/差异化/价格战）
→ 综合竞争强度：__/100（越低越好进入）
```

**③ 精准TAM/SAM/SOM**
- TAM：全量市场（不切实际但说明天花板）
- SAM：可触达市场（按地域/场景/用户群筛选后）
- SOM：**现实可切入市场**（第一年/前期资源下能拿到的份额）
- **关键：说清楚SOM的边界和计算逻辑**

**④ 非对称机会矩阵**
```
              AI解决难度低        AI解决难度高
痛点强度高  ┃ 黄金赛道           ┃ 长期壁垒赛道
            ┃ （优先切入）        ┃ （有实力再做）
痛点强度低  ┃ 红海/同质化        ┃ 伪需求
            ┃ （避开）           ┃ （排除）
```
**把发现的机会点放入对应象限，给出判断依据**

---

### 模块二：竞争诊断

**① 竞争梯队地图**
- 第一梯队（领导者）：核心优势、市场份额/营收、壁垒来源
- 第二梯队（挑战者）：差异化策略
- 新兴力量/跨界玩家：威胁程度

**② 竞品价值链AI拆解**
把竞品的业务链条拆解到环节级别：

| 业务环节 | 成本估算 | 效率 | 用户差评 | AI可替代性 |
|---------|---------|------|---------|-----------|
| [环节1] | 高/中/低 | 高/中/低 | 有/无 | 高/中/低 |

**标注竞品弱点：哪些环节是高成本+低效率+用户差评集中点**

---

### 模块三：用户需求洞察

**① 痛点挖掘链（"抱怨→需求→痛点→价值"转化）**

从搜集到的用户声音中，逐条转化：
```
原始抱怨 → 背后需求 → 核心痛点 → 解决价值 → 付费意愿估算
```

示例转化：
```
"每次手动整理报表太烦" 
→ 自动化数据处理需求
→ 痛点：重复劳动耗时2h/天，出错率10%+
→ 价值：节省人力成本×12个月
→ 付费意愿：企业愿意付500-2000元/月
```

输出 **Top 5 痛点**，每条有：原始证据来源 + 频次评估 + 付费意愿。

**② 用户分层画像（2-3个核心群体）**
每个群体描述：人口属性 / 核心痛点 / 当前方案及不满 / 付费能力 / 决策路径

**③ 蓝海需求（未被满足）**
明确指出哪些需求现有产品没有覆盖或解决极差。

---

### 模块四：投融资温度 + 政策环境

**① 资本信号解读**

| 企业 | 轮次 | 金额 | 时间 | 投资方 | 信号解读 |
|------|------|------|------|--------|---------|

钱在往哪个细分赛道聚集？热度是升还是降？

**② 政策顺逆风判断**
- 支持政策（具体名称 + 力度 + 影响）
- 监管风险（数据合规/行业准入/收紧风险）
- 综合判断：顺风 / 平稳 / 逆风

## 执行注意事项

1. **先搜索再输出**：禁止凭记忆写报告，必须先完成搜索
2. **数据要具体**："市场很大" → 不合格；"2024年市场规模324亿元（来源：艾瑞咨询）" → 合格
3. **机会要接地气**："AI+XX有机会" → 不合格；"用AI客服Agent服务XX行业中小企业，首年目标1000家付费客户" → 合格
4. **脆弱性审计不能省**：每份报告都必须有，这是防止误判的保护机制
5. **标注来源**：每个关键数据点必须注明来源
