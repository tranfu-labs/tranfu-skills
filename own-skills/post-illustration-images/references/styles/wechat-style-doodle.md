# 米色手绘风格的正文配图提示词

> 请基于当前提供的文章内容，生成一套 **4～8 张横版正文配图**，用于长篇文章的正文穿插讲解。
> 目标不是做封面海报，而是做一套**统一视觉风格、便于阅读、帮助解释内容**的文章配图。
> 整套图片需要先对文章内容进行拆解，再按重点生成成套配图。
>
> ---
>
> ## 一、任务目标
>
> 1. 先阅读并理解我提供的文章内容。
> 2. 从文章中提炼出 **4～8 个核心信息点**。
> 3. 每一张图只表达 **1 个核心点**，不要一张图塞太多内容。
> 4. 整套图片风格必须统一，但每张图的：
>
>    * 图标
>    * 版式
>    * 视觉重心
>    * 局部排版
>      都要有**适度变化**，避免机械重复和视觉疲劳。
> 5. 这套图是**正文配图**，不是封面图，所以重点是：
>    **帮助解释、帮助理解、帮助拆解文章内容。**
>
> ---
>
> ## 二、固定视觉风格（必须严格保持）
>
> 整体视觉风格统一为：
>
> **浅米色纸张背景 + 深棕色手绘线条 + 低饱和粉彩色块 + 中文手写感标题 + 简洁图标隐喻 + 科普解释型信息图**
>
> 关键词：
>
> **手绘科普感 / 手账式讲解感 / 温和解释型信息图 / 非科技炫酷风 / 非商务 PPT 风**
>
> 画面气质要求：
>
> * 温和
> * 清楚
> * 克制
> * 有教学解释感
> * 有留白
> * 有条理
> * 像“用手绘图把抽象概念讲清楚”
>
> ---
>
> ## 三、固定画布与尺寸
>
> * 所有图片统一使用 **横版 4:3 比例**
> * 推荐尺寸感：**1600 × 1200**
> * 整套图尺寸必须一致
> * 背景统一，不要忽大忽小，不要尺寸不一
> * 右上角为后置品牌叠加预留区，保持自然留白，不要绘制 logo/占位框/提示文字
>
> ---
>
> ## 四、色彩系统
>
> ### 背景
>
> * 浅米色
> * 奶油白
> * 纸张感暖白
> * 不要纯白，不要强渐变
>
> ### 线条与文字
>
> * 统一使用 **深棕色**
> * 不要纯黑
> * 线条略粗
> * 保留轻微手绘不规则感
>
> ### 辅助色块
>
> 使用低饱和粉彩色，按需搭配：
>
> * 浅桃色
> * 浅粉色
> * 灰蓝色
> * 薄荷绿
> * 淡紫色
> * 米黄色
> * 浅灰色
>
> 这些颜色只作为辅助模块底色、标题底条、局部强调色，不要喧宾夺主。
>
> ---
>
> ## 五、文字风格
>
> 文字必须符合以下规则：
>
> 1. **主标题**
>
>    * 每张图顶部居中或居上
>    * 中文手写感 / 毛笔感 / 手绘感
>    * 深棕色
>    * 简洁、有概括性
> 2. **模块标题 / 小标题**
>
>    * 比正文更醒目
>    * 可以放在色块标题条中
>    * 适合作为当前页的结论句或观点句
> 3. **正文说明**
>
>    * 保持简洁
>    * 每个说明模块尽量控制在 2～4 行
>    * 用通俗表达，不要大段堆字
>    * 尽量让每张图“看一眼就知道这页在讲什么”
> 4. **保留原文核心表达**
>
>    * 不要歪曲原意
>    * 可以适度提炼，但不能乱改核心观点
>
> ---
>
> ## 六、图标与插画风格
>
> 图标统一为：
>
> **深棕色线描 + 简单轮廓 + 少量粉彩填充**
>
> 图标要有“教学隐喻”作用，不只是装饰。
>
> 常用图标/元素可根据内容灵活选用：
>
> * 问号 / 对话框
> * 人物 / 角色头像
> * 文件 / 清单 / 说明书
> * 勾选框 / checklist
> * 放大镜 / 地球 / 上下文符号
> * AI 方框 / 机器人小图标
> * 工具箱 / 漏斗 / 路线箭头
> * 天平 / 书本 / 示例页
> * 编号步骤 / 流程箭头
> * 卡片 / 告示牌 / 模块框
>
> 图标要和本页内容一一对应，不要乱用。
>
> ---
>
> ## 七、整套图片的结构规则
>
> 这是一套正文配图，所以请先根据文章内容自动拆成 **4～8 页**。
> 拆解时遵循以下原则：
>
> 1. **每页只讲一个核心点**
> 2. **整套页序有逻辑递进**
> 3. 常见顺序优先参考：
>
>    * 问题 / 误区
>    * 原因 / 本质
>    * 方法 / 结构
>    * 示例 / 判断标准
>    * 操作步骤 / 应用建议
>    * 最后结论 / 关键提醒
> 4. 如果文章适合分层讲解，可拆成：
>
>    * 第 1 张：总问题 / 总结论
>    * 第 2～5 张：核心拆解点
>    * 第 6～7 张：方法 / 判断标准
>    * 第 8 张：收束提醒 / 一句话结论
> 5. **整套页面不重复，如果不够 8 张就压缩成 5～6 张，以精简原则为主。**
> ---
>
> ## 八、版式变化规则（整套统一但不重复）
>
> 整套图片保持同一视觉系统，但请在 4～8 张图中交替使用以下版式，让它们看起来是一套，但不完全一样。
>
> ### 版式 A：四宫格 / 多模块解释
>
> 适合：
>
> * 四个原因
> * 四个要点
> * 四种误区
> * 四个判断标准
>
> 特点：
>
> * 顶部标题
> * 中间 2×2 模块
> * 每个模块带图标和说明
>
> ### 版式 B：中央主视觉 + 标签说明
>
> 适合：
>
> * 流程
> * 转化关系
> * 分层结构
> * 输入到输出
>
> 特点：
>
> * 中央一个主图形（如漏斗、流程、结构图）
> * 周围或右侧标签说明
> * 主视觉必须居中，画面平衡
>
> ### 版式 C：中央大物件 + 四周说明
>
> 适合：
>
> * 一个核心概念的不同侧面
> * 一个方法的用途
> * 一个工具的作用
>
> 特点：
>
> * 中下部有一个大隐喻插画（如工具箱、文件夹、工作台）
> * 左右或四周分布说明模块
>
> ### 版式 D：步骤流程页
>
> 适合：
>
> * 操作顺序
> * 方法步骤
> * 先后逻辑
>
> 特点：
>
> * 编号 1/2/3/4
> * 箭头连接
> * 每步可搭配小图标
>
> 要求：
>
> * 同一套图中至少使用 **2～3 种不同版式**
> * 不要每一页都长得完全一样
> * 但色彩、线条、字体、气质必须高度统一
>
> ---
>
> ## 九、排版要求
>
> * 主视觉要居中或视觉重心平衡
> * 四周留白要充足
> * 不要拥挤
> * 不要文字挤压
> * 不要图标过大压住文字
> * 不要出现明显偏左或偏右失衡
> * 每张图都要有“标题—图形—说明”清晰层级
> * 保证每页一眼能扫懂重点
> * 右上角品牌预留区保持自然留白，不要画占位框、边框、贴纸或提示文字
>
> ---
>
> ## 十、输出内容要求
>
> 请生成一整套 **4～8 张正文配图**，并满足：
>
> 1. 每张图围绕文章一个核心点
> 2. 每张图都要有：
>
>    * 标题
>    * 图标 / 隐喻插画
>    * 简短说明
> 3. 所有图片风格统一
> 4. 每张图片图标与排版略有变化
> 5. 保持整套阅读顺序清晰
>
> ---
>
> ## 十一、严格负面约束
>
> 不要出现以下情况：
>
> * 不要赛博科技感
> * 不要蓝紫霓虹
> * 不要商务 PPT 风
> * 不要极简冷淡风
> * 不要真实照片感
> * 不要 3D 渲染感
> * 不要高饱和颜色
> * 不要复杂花纹背景
> * 不要随机英文
> * 不要二维码
> * 不要条形码
> * 不要水印
> * 不要图标风格混乱
> * 不要把一页塞太满
> * 不要改变原文核心意思
>
> ---
>
> ## 十二、最终目标
>
> 最终效果应该像：
>
> **一套可直接穿插进长文中的手绘科普信息图**
> **统一、克制、清楚、可读，有解释感，不是宣传海报。**
>
> 核心原则只有一句：
>
> **这套图的本质，是“用统一的手绘信息图视觉系统，把长文内容拆开讲清楚”。**
