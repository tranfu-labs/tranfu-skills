# 文章配图生成

给公众号 / 小红书 / 知乎文章一次配齐整套插图, 从读懂正文到分镜再到 QA 全程兜底。

## 什么时候用它

**公众号正文配图**:

我写完一篇公众号推文, 粘正文说「帮我做几张配图」, skill 读全文按方法 / 流程 / 对比 / 清单等表达需求选风格, 一张一张出。

**小红书组图**:

要一组封面 + 正文轮播, 期望整套视觉统一、每张单独承载一个核心含义, skill 按锚点排分镜再逐张出图。

**指定风格 ID**:

我点名 `wechat-doodle` / `xhs-explainer-notebook` / `zhihu-tech`, skill 直接跳过风格采访, 按对应 `style_file` 与 `style_spec` 出。

**按表达需求点单**:

我按段落说「这段要一张流程图, 那段要一张常见错误图」, skill 按锚点排分镜, 一图一意, 不硬凑数量。

**续做与叠 logo**:

上一次跑完的裸图想补 logo / 加一张 / 单张重跑, 只动指定图, 绝不重跑整套。

**不接**:

纯摄影 / 人像修图 / 产品渲染 / 写实品牌大片 → 不触发; 图内要放整段精确长文 → 不触发 (原生图模型画不稳); 明确点名走另一个图像 skill → 让那个 skill 接。

## 它会产出什么 / 你会看到什么

**默认一次只生成一张图, 整套统一风格靠 `style_spec` 兜底, 品牌 logo / 页码 / 占位框绝不由图像模型画**——这三点最反常识。

- **落盘位置**: `post-illustration-output/<content-slug>/` 落项目根, 绝不写进 skill 目录
- **每次产出**: `shot-list.md` (分镜) + `prompts/*.md` (每张单独一份) + `manifest.md` (整套元数据 YAML)
- **图片路径**: `images/unbranded/*.png` (裸图) + `images/branded/*.png` (叠完 logo); 品牌关闭时只出 `images/*.png`
- **叠加副作用**: 会调 `scripts/apply-brand-overlay.mjs` 做叠加, 依赖 `rsvg-convert`; 首次缺依赖会问一次要不要装, 拒装则回落裸图交付
- **QA 循环**: 单张失败最多重跑 2 次, 仍不过则退回锚点调整, 或征你同意留 `residual_risk` 交付
- **绝不会做**: 让图像模型画 logo / TF / 水印 / 页码框; 一次生成整套轮播; 复制 `style_reference` 的语义内容

## 前置条件 / 边界

**前置**:

Codex / OpenAI 原生图像生成能力可用; 目标平台是公众号 / 小红书 / 知乎; 启用品牌叠加时机器上需有 `rsvg-convert` (脚本会先检测, 拒装可回落无品牌)。

**不接的场景**:

- 摄影 / 人像修图 / 产品渲染 / 写实品牌大片
- 图内必须精确放整段长文的场景 (原生图模型不稳)
- 显式要走另一个图像生成 skill 的请求

**微妙边界**:

- 用户说「给我做 5 张」: 5 是**上限而非配额**, 锚点不够宁可少交也不硬塞凑数图
- `style_reference` 是 QA 用的**长期基线**, 绝不作为生成输入, 也不复制其语义
- 品牌槽位三档: 用户显式关 > `style_spec` 未开 > 默认关; 任一命中即不叠加
- 续做单张: 只动指定图, 不重跑整套; 无裸图底图时会问是否接受重复贴 logo 的风险
