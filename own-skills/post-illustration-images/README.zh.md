# 多平台文章配图

用一套已登记的视觉风格，把成稿转成统一、可追溯的平台图片组。

## 什么时候用它

**为文章制作配图**

我有一篇公众号、知乎、微博或头条号成稿，希望选出最值得视觉化的内容，并逐张生成和检查。

**制作内容组图**

我有一篇小红书笔记，需要协调一致的封面和正文组图，而不是一组彼此无关的装饰图片。

**续做现有图片**

我想给已有图片包追加、恢复、重新加品牌或重做其中一张，不希望重跑已经验收的其他图片。

**不接**

它不处理照片精修、产品样机、写实品牌广告、必须准确呈现长文案的图片，或用户已点名的其他出图能力。

## 它会产出什么

**整组只选一次风格，每张图片只表达一个核心意思。**

- **规划文件**：保存内容分析、必需的分镜清单，以及每张图独立的提示词
- **图片产物**：逐张生成并检查位图，保留通过验收的原生像素
- **品牌处理**：只有规则明确启用时，才用真实品牌素材进行确定性叠加
- **图片包记录**：在 `manifest.md` 记录后端、尺寸、尝试次数、路径、验收和剩余风险
- **文件改动**：在用户项目中创建或更新 `post-illustration-output/<content-slug>/`
- **外部动作**：使用一个通过核实的图片后端，可能产生生成费用
- **绝不会做**：让模型绘制 Logo、强行拉伸合格图片，或把生成产物写进能力目录

## 前置条件与边界

**前置条件**

需要可读的正文内容和目标平台，支持公众号、小红书、知乎、微博和头条号。对应平台必须有已登记风格，并且图片生成后端已经通过核实。

**相邻能力分工**

| 需求 | 交给 |
|---|---|
| 构建可复用风格 | **visual-builder** |
| 只做公众号封面 | **wechat-sketch-cover** |
| 运行完整生产流程 | **content-production** |

**不接的场景**

- 自然摄影、人像、产品精修或广告画面的精确复刻
- 必须在图片中准确呈现长篇文字
- 没有已登记风格，且用户未确认备用方案的平台

**容易混淆的边界**

- 用户指定的数量是上限，不代表可以补无意义的图片
- 是否加品牌先看用户明确要求，再看所选风格的默认规则
- 比例不符时会按已核实尺寸重试，不会缩放合格图片来强行过关
