# 06. 批量信息的判断效率 / Batch Information Judgment Efficiency

这是一条**审查维度**，按操作手册使用：先由 `00-inspection-procedure.md` 把页面探寻成证据包，再用本文件的形式目录逐条比对、判定。本文件**不写探寻机制**（怎么截图 / 怎么点开状态都在 `00`），只写：① 解决什么问题；② 向引擎申请的额外探查；③ 形式目录 + 判定。

边界：数据 loading / 空 / 无结果 / 错误 / 权限 / 部分数据这些状态本身归 `05`，这些状态的文案清晰度归 `13`；任何状态导致的布局移动归 `04`；即时可操作性 / 操作反馈归 `03`，动作完成后的结果反馈归 `12`；视口 / 容器重排让多记录视图挤坏归 `11`；用户在产品层级中的当前位置、导航范围和返回路径归 `08`；同类元素 / 状态的视觉一致性归 `09`。本维度只主责**多条相似记录的横向比较、异常扫描、数据范围识别，以及批量 / 行内 / 页级 / 全局操作的作用范围分辨**。

## 一、解决什么问题

用户面对一屏多条同类记录（表格、列表、卡片列表、搜索结果、看板、时间线）时，能不能**快速判断、不被迫逐行细读**：横向比出金额 / 时间 / 状态 / 数量 / 负责人 / 风险 / 进度等关键字段的差异；在一片普通记录里一眼挑出失败 / 过期 / 风险 / 待处理项；分清某个操作作用于单行、选中项、当前页、全部结果还是整个列表；知道自己正在看的是全部数据、搜索结果、某筛选条件、某分页还是排序后的视图；区分 `0`、空值、未知、未填写、不适用、加载失败这几种本不相同的缺失。

## 二、向探寻引擎申请的额外探查（→ `00` 环节二）

引擎在池化执行时，为本维度额外做（以下都不产生副作用，只读可做；需要副作用才能到达的状态记成「声明了但只读被阻挡」缺口，挂本维度名下，输出 `blocker`）：

- **真实数据密度下的列 / 字段盘点**：用页面真实数据（含长名称、长说明、长负责人、长状态原因）盘点多记录视图里有哪些关键字段、它们的对齐方式；对文字最长的那几行 / 单元格做**裁剪放大重采**，看长文本是否把关键字段挤偏 / 换行 / 推错位（不靠 `getClientRects().length` 行数——它对 `td` 等单元格恒为 1，内部换行测不出；`scrollWidth > clientWidth` 只判横向裁切、不判换行撑高，仅作佐证）。
- **关键字段可横向比较性**：标出哪些字段是用户判断差异 / 异常 / 优先级的关键字段，盘点它们是埋在自然段里、还是排成可对位扫描的列 / 字段；记录数值字段的对齐是否一致。
- **异常项可发现性**：盘点失败 / 过期 / 危险 / 待处理 / 阻塞等记录，相对普通记录有没有颜色 / 图标 / 标签 / 分组 / 排序 / 位置等可被一眼扫到的区分线索。
- **操作入口与作用范围盘点**：盘点每个操作入口（行内按钮、批量按钮、页级按钮、全局按钮），记录它的视觉归属与声明的作用对象（单行 / 选中项 / 当前页 / 全部结果 / 整列表）。
- **范围状态控件盘点**：盘点筛选 / 排序 / 搜索 / 分页 / 选择控件，记录当前条件、排序状态、页码、结果范围、已选数量是否在界面上可见。只读够得到的视图（已有筛选态、已有分页态、URL 带 query 的结果页）正常审；需要真去提交筛选 / 翻页副作用才能到达的，记 `blocker` 缺口。
- **缺失值表现盘点**：盘点 `0` / 空 / `-` / `N/A` / 未知 / 未填写 / 不适用 / 加载失败的单元格表现，记录它们是否被混成同一种写法。

## 三、判定的两道闸（贯穿所有分类）

1. **形式目录挂嫌疑（高查全）**：宁可错抓，命中任一形式 = 嫌疑。
2. **判定定罪（高查准）**：只有"会让用户**比不出差异、找不到异常、误判数据范围、搞错操作作用对象，或把不同含义的缺失当成一回事**"才定罪；否则按各分类的**释放理由**放过。够不到的范围 / 状态一律 `blocker` 缺口，不猜测、不放过。

本维度不做样式聚类，没有显眼 / 细微轴；每个分类的嫌疑信号与定罪测试各自写在判定里。

## 四、形式目录 + 判定

### A. 关键字段比不出差异

形式：

- A1 多记录做成一串信息结构相同的漂亮卡片，金额 / 时间 / 状态 / 数量等关键字段埋在自然段里，无法横向对位。
- A2 数值字段对齐不一致或混在文字中，用户得逐个找数字才能比大小。
- A3 关键字段被长文本（长名称 / 长说明）挤偏、换行、推错位，破坏对位扫描。
- A4 其它让用户必须逐行读完整段文字才能比较关键差异的情况。

判定：用探查盘点的关键字段与对齐 → 用户是否得逐行细读才能比出差异 = 嫌疑 → 在整页截图 + 最长行 / 单元格的放大特写上确认关键字段确实不能对位横向比较、或被长文本挤偏 / 换行 / 推错位 → 定罪。释放：不靠表格也行，列表 / 卡片 / 看板 / 时间线只要支持关键字段对位比较即可；数值右对齐、截断、tooltip 只是手段，不强制。

### B. 异常项扫不出来

形式：

- B1 失败 / 过期 / 风险 / 待处理 / 阻塞记录显示得和普通记录几乎一样，得逐行找。
- B2 异常只靠一处极弱的线索（如灰字一行），在整片记录里扫不到。
- B3 多种异常语义混在一起、强度不分，用户分不清哪条更要紧。
- B4 其它让用户必须逐行读才能发现异常 / 待处理项的情况。

判定：用探查盘点的异常项区分线索（颜色 / 图标 / 标签 / 分组 / 排序 / 位置）→ 普通记录里能否一眼扫到异常 = 嫌疑 → 截图确认用户得逐行寻找才发现 → 定罪。释放：异常有任一可被快速扫到的清晰线索即可，不强制用 badge。

### C. 操作作用范围分不清

形式：

- C1 行内按钮、批量按钮、页级按钮、全局按钮视觉上混在一起，看不出作用于单条、选中项、当前页还是全部。
- C2 批量操作没说明作用于"选中项"还是"全部结果"，已选数量也不可见。
- C3 全局 / 危险操作（清空、全部导出、批量删除）和行内操作长得差不多，易误把单行操作当成全量。
- C4 其它让用户搞不清某个操作影响范围的情况。

判定：用探查盘点的操作入口视觉归属与声明作用对象 → 视觉归属 ≠ 实际作用范围、或作用范围无任何说明 = 嫌疑 → 截图确认用户会误判这个操作影响单条还是全量 → 定罪（误解作用范围直接造成错误操作，定罪从严）。释放：作用对象有清晰线索（位置归属 / 标签 / 已选计数 / 确认文案）即可，不强制特定布局。

### D. 数据范围误判

形式：

- D1 筛选 / 搜索后只显示结果列表，不保留当前条件，用户误以为这就是全部数据。
- D2 分页只显示当前页内容，无页码 / 总数 / 结果范围，用户误判"数据没了"。
- D3 排序 / 分页 / 选择状态同时存在时，不说明当前视图，用户对一部分数据做了全量判断。
- D4 其它让用户分不清在看全部数据、搜索结果、某筛选、某分页还是排序后视图的情况。

判定：用探查盘点的范围状态控件 → 当前条件 / 排序 / 页码 / 结果范围是否可见 → 缺失 = 嫌疑 →（只读够得到就截图确认，够不到记 `blocker` 缺口）确认用户会误判数据范围 → 定罪。释放：界面持续显示当前条件、排序状态、页码或结果范围即可。

### E. 缺失值被混成一种表现

形式：

- E1 `0` 和"没有数据"写成同一种表现，业务含义被抹平。
- E2 空值、未知、未填写、不适用、加载失败全写成 `-` / `N/A` / 空白，无法区分。
- E3 无权限 / 部分数据未返回的缺失，和"确实没有"长得一样。
- E4 其它让用户分不清缺失到底是哪种含义的情况。

判定：用探查盘点的缺失值表现 → `0` / 空 / 未知 / 未填写 / 不适用 / 加载失败是否被混成同一种写法 = 嫌疑 → 截图确认用户会基于错误含义做判断（把 `0` 当没数据、把无权限当没有）→ 定罪。释放：不同含义的缺失有可区分的表现即可（缺失态本身的 loading / 空 / 错 / 权限归 `05`，其文案清晰度归 `13`；本类只主责这几种缺失是否被混成一种）。

## 五、边界 / 非目标

- 不要求必须使用表格：列表、卡片列表、看板、时间线都可以满足本维度，只要支持批量判断。
- 不把"数值右对齐""状态用 badge""长文本截断 / tooltip""行距 / 分割线 / 冻结列 / sticky header / 批量操作栏 / 列宽策略"写成 MUST；它们是实现手段，只在相应数据规模和操作场景下作为选择。
- 不把"视觉密度要服务扫描"单列成 MUST；密度是方向不是可验证结果，本维度验证的是差异、异常、范围、作用对象是否容易判断。
- 数值阈值（如"卡片字段超过几个就埋住关键字段"）一律作为嫌疑信号，不当硬性 MUST / token。
- 不把内部数据建模、组件抽象、表格库选择、token 使用方式当成本维度目标；只检查用户能看到、感到、被影响的批量判断效率。
- 不主责以下，按归属路由：
  - 数据 loading / 空 / 无结果 / 错误 / 权限 / 部分数据状态本身 → `05`；这些状态的文案清晰度 → `13`。
  - 任何状态导致的布局移动 → `04`；即时可操作性 / 操作反馈 → `03`；动作完成后的结果反馈 → `12`。
  - 视口 / 容器重排让多记录视图挤坏 → `11`。
  - 用户当前位置、导航范围、返回路径 → `08`；同类元素 / 状态的视觉一致性 → `09`。
