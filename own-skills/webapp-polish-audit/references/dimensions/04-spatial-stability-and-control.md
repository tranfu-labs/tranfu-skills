# 04. 布局稳定的用户掌控感 / Spatial Stability and User Control

这是一条**审查维度**，按操作手册使用：先由 `00-inspection-procedure.md` 把页面探寻成证据包，再用本文件的形式目录逐条比对、判定。本文件**不写探寻机制**，只写：① 解决什么问题；② 向引擎申请的额外探查；③ 形式目录 + 判定。

边界：即时可操作性 / 操作反馈归 `03`，动作完成后的结果反馈归 `12`；数据 loading / 空 / 无结果 / 错误 / 权限 / 部分数据等状态本身是否有解释和下一步归 `05`，这些状态的文案清晰度归 `13`；表单字段校验 / 保留 / 提交信心归 `07`；视口 / 容器重排归 `11`；视觉一致性归 `09`。本维度只主责**任何状态变化导致的布局移动**——内容是否塌陷、控件是否在操作过程中位移，让用户丢失阅读位置或误点。

## 一、解决什么问题

用户在状态变化时能不能**保住空间方向感和操作安全**。页面从 loading 到 loaded、翻页、筛选、排序、刷新、校验、删除后列表收缩，以及 toast / alert / drawer / modal / 展开收起出现时，正在看的区域不应突然塌陷或大幅跳动，正准备点的控件不应突然移走——否则用户会丢失阅读位置、操作目标跑掉、误点相邻（尤其危险）按钮，或感到页面在状态变化时不稳定、不可控。

## 二、向探寻引擎申请的额外探查（→ `00` 环节二）

引擎在池化执行时，为本维度额外做（以下都不产生副作用，只读可做；需要副作用才能到达的状态记成「声明了但只读被阻挡」缺口，挂本维度名下，输出 `blocker`）：

- **loading→loaded 前后对比**：捕捉同一区域从加载态（spinner / skeleton）到内容态的布局盒，前后对比尺寸与位置；记 skeleton 与真实内容的盒尺寸差。
- **分页 / 筛选 / 排序前后对比**：对表格 / 列表执行只读可达的翻页、筛选、排序、搜索，记录满页 / 少量数据 / 空页 / 加载中 / 错误页下，表格区域、分页器、操作栏、后续内容的布局盒位移（前后对比）。
- **校验前后对比**：触发只读可达的 inline 校验 / 错误提示出现，记录行高、按钮、容器布局盒是否被撑动（提交类副作用记缺口）。
- **交互反馈前后对比**：对 hover / pressed / selected / focus 出现的态，记录按钮、行、toolbar、卡片的布局盒尺寸前后变化。
- **toast / alert / drawer / modal / 展开收起前后对比**：到达只读可达的这些态，盘点出现前后的整页布局盒位移，重点记是否把正在操作的区域推走、是否让相邻控件位移。
- **盘点布局盒几何**：对上述每个前后对，记录关键控件的包围盒（位置 + 尺寸），供位移量比对。

## 三、判定的两道闸（贯穿所有分类）

1. **形式目录挂嫌疑（高查全）**：宁可错抓，命中任一形式 = 嫌疑。
2. **判定定罪（高查准）**：只有"会让用户**丢失阅读位置、操作目标移动、误点，或感到页面在状态变化时不稳定**"才定罪；否则按各分类的**释放理由**放过。够不到的状态一律记 `blocker` 缺口，不猜测、不放过。

本维度不做样式聚类，没有显眼 / 细微轴；每个分类的嫌疑信号与定罪测试各自写在判定里。

## 四、形式目录 + 判定

### A. 内容状态切换让用户丢失位置

形式：

- A1 loading → loaded、empty → data、data → error、刷新时，正在看的核心区域突然塌陷或大幅上下跳。
- A2 状态切换后，原本可见的内容被挤出视口，用户得重新找位置。
- A3 错误 / 空态以与内容态差异极大的尺寸渲染，导致状态间反复跳动。
- A4 其它让用户在内容状态切换时丢失阅读位置的布局移动。

判定：用探查的 loading→loaded / 状态切换前后对比 → 核心区域布局盒位置 / 尺寸是否大幅变化 → 是 = 嫌疑 → 截图前后对比，确认用户正在看的区域确实塌陷或跳动、需要重新找位置 → 定罪。释放：某些内容加载和响应式变化天然会动，不要求零 layout shift。

### B. 表格 / 列表翻页时阅读区域忽高忽低

形式：

- B1 从满页翻到少量数据 / 空页 / 加载中 / 错误页时，表格区域高度塌陷。
- B2 分页器、批量操作栏随表格高度变化一起上下跳。
- B3 翻页后下一段内容（页脚 / 相邻区块）位置漂移，用户的视线锚点丢失。
- B4 其它让翻页 / 筛选 / 排序后阅读区域或分页控件位移的情况。

判定：用探查的分页 / 筛选 / 排序前后对比 → 满页 / 少量数据 / 空页 / 加载中 / 错误页下，表格区域、分页器、操作栏、后续内容的布局盒是否忽高忽低 → 是 = 嫌疑 → 截图前后对比，确认分页器或操作按钮确实跳动、用户阅读位置不稳 → 定罪。释放：不要求复杂虚拟列表一次性解决所有高度问题，只要求用户可感知的关键区域稳定。

### C. 加载态与完成态空间占用差距过大

形式：

- C1 loading 只放一个小 spinner，真实内容出现后整页突然撑开。
- C2 skeleton 尺寸和真实内容差很多，先跳一次 skeleton、再跳一次真实内容（二次跳动）。
- C3 占位区高度远小于实际内容，内容到达瞬间把后续元素推下去。
- C4 其它让加载态与完成态空间占用明显错配的情况。

判定：用探查的 loading→loaded 前后对比 → 加载态（spinner / skeleton）与内容态的布局盒尺寸差是否明显 → 是 = 嫌疑 → 截图前后对比，确认页面确实先出现小占位再突然撑开、或出现明显二次跳动 → 定罪。释放：`min-height` / placeholder rows / skeleton / reserved space 都只是手段，不强制某一种；尺寸接近即可放过。

### D. 交互反馈改变布局结构

形式：

- D1 hover / selected / focus 通过加粗、加边框、改尺寸让行高或按钮宽度抖动。
- D2 validation message / 提示文本出现时把相邻元素推走，而非预留空间。
- D3 pressed / 状态切换让 toolbar、卡片宽度、容器尺寸明显变化。
- D4 其它让交互反馈改变布局结构、而非纯视觉变化的情况。

判定：用探查的交互反馈 / 校验前后对比 → hover / pressed / selected / focus / error / validation message 出现时，按钮、行高、toolbar、卡片、容器的布局盒尺寸是否明显变化 → 是 = 嫌疑 → 截图前后对比，确认这不是明确设计过的展开行为、确实造成抖动 → 定罪。释放：明确设计过的展开 / 多行行为；纯视觉变化（颜色 / 阴影，不改盒尺寸）。

### E. 可点位置在操作过程中突然移动

形式：

- E1 保存中 / 加载中 / 分页中，用户原本要点的按钮移走。
- E2 删除后列表收缩，相邻行的危险操作（删除等）滑到光标下，造成误点风险。
- E3 toast / alert 出现把内容推动，正准备点的控件位移或消失。
- E4 其它让关键 / 危险控件在操作过程中突然移动的情况。

判定：用探查的删除后 / 加载中 / toast 出现前后对比 → 用户可能点击的关键 / 危险控件布局盒是否在操作过程中位移 → 是 = 嫌疑 → 截图前后对比，确认控件确实移到危险位置或消失、用户会误点相邻或丢失目标 → 定罪。释放：控件位置在操作过程中保持稳定，或移动幅度小且无危险相邻项。

### F. 大变化区域破坏空间关系

形式：

- F1 展开筛选面板 / 抽屉 / 弹窗时，页面突然重排，用户看不出原内容与新区域的关系。
- F2 展开 / 折叠会移动内容的区域，把后续内容大幅推走且无可理解的过渡。
- F3 drawer / modal 关闭后页面布局与打开前不一致，用户失去方向感。
- F4 其它让大变化区域破坏空间关系、用户无法理解发生了什么的情况。

判定：用探查的 drawer / modal / 展开收起前后对比 → 出现 / 关闭时页面是否突然重排、原内容与新区域关系是否可读 → 不可读 = 嫌疑 → 截图前后对比，确认用户无法判断原内容、新内容、操作目标之间的空间关系 → 定罪。释放：可以动，只要用户能理解发生了什么、空间关系清楚。

## 五、边界 / 非目标

- 不要求所有页面完全零 layout shift：某些内容加载和响应式变化天然会动。
- 不把某种 CSS 技术写成 MUST：`min-height`、placeholder rows、skeleton、reserved space 都只是手段。
- 不要求为稳定高度牺牲信息展示，例如强行把所有卡片截断到不可读。
- 不要求复杂虚拟列表一次性解决所有高度问题；只要求用户可感知的关键区域稳定。
- 不把 CLS 指标本身当唯一标准；这里关心用户操作中的掌控感，不以单一数字替代人工前后对比。
- 不把内部状态管理、组件抽象或性能实现当本维度目标；只检查用户能看到、感到、被影响的空间稳定结果。
- 即时可操作性 / 操作反馈归 `03`，动作完成后的结果反馈归 `12`。
- loading / 空 / 无结果 / 错误 / 权限 / 部分数据等状态是否有解释和下一步归 `05`，这些状态的文案清晰度归 `13`。
- 表单字段校验 / 保留 / 提交信心归 `07`；视口 / 容器重排（窄屏重排、横向滚动等）归 `11`；同类元素 / 状态的视觉一致性归 `09`。
