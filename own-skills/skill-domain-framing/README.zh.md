# Skill 任务域框定

给还没成型的原始素材 (事故复盘 / guardrail / PR / issue / 项目文档) 框 skill 容器名, 4+ 候选打分让触发的词对得上, 只出决策, 绝不写 SKILL.md。

## 什么时候用它

**事故复盘沉淀**:

我手里有一份事故复盘, 想把纠偏经验固化成 skill, 但不知道容器该叫什么, 也不想让事故里的缓存 / 供应商 / 文件名把容器名带偏。

**讨论型开场**:

「这个该做成什么 skill」/「先帮我定一下范围」——想在动 SKILL.md 之前, 先把边界、成功信号和抽象方向锚清楚。

**检查候选名称**:

我已经想好叫 X, 想让评分表挑战一下——名字扛得住就用, 冒出更高分的候选就说服我换。

**存量素材沉淀**:

`AGENTS.md` 里一段 guardrail、一份 PR 讨论, 或者一个 issue 线程——想让 skill 从噪声里抬出对的容器名, 而不是拿我随手打的名词直接凑。

**判定通过后承接**:

`skill-content-fit` 刚判了通过, 需要先锁容器名并把素材放进对的层 (主流程 / 验证 / 排障), 才轮到 `skill-create-workflow` 上场。

**不接**:

输入已经是 skill / `SKILL.md` / 已装 skill → **skill-improve-workflow**; 判断素材够不够格沉淀成 skill → **skill-content-fit**; 起中英文成对显示名 → **skill-name-generation**; 定完容器后写 `SKILL.md` 正文 → **skill-create-workflow**。

## 它会产出什么

**只出框架决策, 绝不写 SKILL.md——最反常识的一点。**

- **候选评分表**: 至少 4 个候选, 至少覆盖 3 个抽象维度 (用户结果 / 重复任务 / 动作·编排 / 实现对象 / 平台 / 代码位置); 每候选 3 维 × 每维 0-2 分 (触发对齐 / 边界清晰 / 路径分层); 每行配一句评语, 解释非满分维度的扣分理由。
- **三行汇总**: `Top1` / `Top1-Top2 分差` / `用户指定候选`——下游 `skill-create-workflow §4` 会机械读这三行做路由判断。
- **范围边界**: 包含 / 排除 / 放置三行, 说清楚容器管什么、不管什么、源经验落在主流程 / 分支 / 验证 / 故障排查哪一层。
- **路径分层**: 正常 / 验证 / 排障三层——绝不让事故里的排查动作 (探针页 / 日志查询 / 缓存绕过 / 回滚) 冒充主工作流。
- **触发条件与验收标准草稿**: 一段人话, 下游 skill 编写者可以直接抬进 SKILL.md。
- **绝不会做**: 写 `SKILL.md`、动仓库任何文件、把用户指定的容器名自动当选——用户提的名字也得靠分数排上去, 低分只用来反驳, 不用来偷偷丢掉。

## 前置条件 / 边界

**前置**:

一段能读的原始素材——事故复盘 / guardrail 文档 / issue 或 PR 讨论 / `AGENTS.md` 片段 / 项目文档都行。素材里至少能看到 (1) 一次真实失败, (2) 一条规则, 或 (3) 一个工作流三者之一; 都看不到时最多问 3 个澄清问题, 再缺就停在阻塞态, 绝不硬凑。

**相邻 skill 分工**:

| 动作 | 交给 |
|---|---|
| 判断素材够不够格沉淀成 skill | **skill-content-fit** |
| 定完容器后写 `SKILL.md` 骨架 | **skill-create-workflow** |
| 起中英文成对显示名 | **skill-name-generation** |
| 审 / 改进已有 skill | **skill-improve-workflow** |

**不接的场景**:

- 输入本身已经是 skill / `SKILL.md` / 已装 skill 的内容
- 给一个已有 skill 换英文目录名
- 起中英文成对显示名
- 审 / 修既有 skill

**微妙边界**:

- 用户指定的容器名不自动当选——照样进评分表打分, 低分用来反驳, 而不是悄悄丢掉
- 动作 / 编排型的动词名 (`create` / `review` / `improve`) 是合法一等落点, 不因「是动词」被扣分——真正的判据是「用户请求这能力时会不会用这个词」, 而不是名字长得像动词还是名词
- 素材撑不出 4 候选 × 3 维度 → 阻塞而不硬凑, 让用户补素材再回来
