# Editorial Policy v2

## 优先级

1. 已核验事实、安全与合规边界。
2. 当前平台规范。
3. 中性核心画像、平台画像和本篇细分受众。
4. B 风格。
5. 标题、视觉和表达偏好。

低优先级约束不得扩大 claim、强化因果、删除限定或补写数字、经历、引语、人物背书和产品效果。

## A/B 隔离

- A/B 共享同一 claims、evidence map、control outline、模型和参数。
- A 禁止读取 B 风格快照或 B 产物。
- B 必须绑定本次 `style-b.md`，只改变表达与可授权的叙事顺序。
- 两者先生成平台中立母稿；五平台稿直接从同版本母稿适配。
- provenance 必须证明输入路径与哈希，不接受回复中的自我声明替代。

## 画像合并

顺序固定：`core_audience -> platform_overlay -> article_segment`。后层只能细化，不得无证据替换年龄、职业、动机或痛点。平台分发数据不能传播到中性核心画像或其他平台。

## 平台边界

| 平台 | 主要目标 | 禁止做法 |
|---|---|---|
| 公众号 | 完整叙事、清晰层级、连续阅读 | 用夸张标题替代论证 |
| 小红书 | 高扫描性、场景、步骤和收藏价值 | 虚构体验、堆标签、照搬长文 |
| 知乎 | 问题导向、证据、反方和边界 | 伪装权威、只有结论没有论据 |
| 微博 | 单条自洽、事实前置、紧凑 | 删节造成失真、无关话题词 |
| 头条 | 背景完整、事实清楚、收益明确 | 扩大承诺、只改公众号标题 |

允许调整开头、篇幅、段落密度、信息顺序、互动方式和平台格式。禁止调整事实、数字、引语、效果、适用范围、因果强度和作者真实经历。

## 三轮审校

1. 逻辑事实：核对主旨、证据、claim 范围、反方、限制和内部一致性。
2. 平台文风：去模板腔、机械排比、空泛过渡和重复，但不虚构真人细节。
3. 细节格式：错别字、标点、术语、标题层级、链接、标签和 Markdown。

proofreading provider 负责三轮语义审校；总控以原始平台 `draft.md` 为基线独立执行两次 claim
regression，不能把 verified claims 当作新增内容许可。独立 semantic review 必须检查新增结论、范围
变化、因果增强、事实新增、事实遗漏和专名漂移。任一失败必须回滚或返回 editing 阶段；需要补事实时
再回到研究阶段。

Provider 在单稿 PASS 前先运行同一 `markdown-alignment` 自动检查。正文按 Markdown 小节和结构类型
一对一对齐，H1 不参与正文限定词、引语或效果匹配；字节相同直接 PASS，对齐歧义必须阻断。

## 标题

- 只读取十份稳定终稿。
- 每个标题映射终稿文件中逐字存在的文本承诺锚点，事实门高于数量和吸引力；交付候选全部为 `PASS`。
- 候选使用 `rank`、`recommended` 和 risk；内部数字 score 保持 provider 私有。
- 数量固定为公众号 3、知乎 3、小红书 5、微博 2、头条 4，每个版本分别计算，总计 34。
- 微博单行 hook 存在 `title`，唯一 `#...#` 话题词单独存在 `topic_phrase`；其他平台该字段为 null。
- 微博发布包唯一 H1 只使用 hook，不能把 `topic_phrase` 拼入 H1。
- 入选标题必须成为发布包 Markdown 的唯一 H1；不得只写 metadata。
- 公众号入选标题还必须符合封面输入：含汉字、单行、2-35 个非空白字符。

## 视觉、封面与优化

- 只为五份入选稿生成正文配图；落选版本不进入视觉阶段。
- plan 与 generate 分离，计划绑定正文哈希；generate 不得改变已决定锚点。
- 图片不得制造正文没有的数据、界面、成绩、背书或产品效果。
- 公众号发布封面由 `wechat_cover` 独立生成，不插入正文、不进入正文 manifest。
- 封面标题必须由 provider 视觉检查观察为完全一致并绑定最终图片哈希；未执行确定性 OCR 时必须如实记录。`BEST_EFFORT` 不可交付。
- 正文图优化不得覆盖生成源。优化结果更大时保留原图；封面保持无损 PNG 和固定尺寸。
- 只有压缩 candidate 严格更小时才能进入发布包；相等或更大时发布图必须与原图字节一致。JPEG 原图回退保留为 `.jpg`，不得伪装成 WebP。

## 排版

- 图片和入选标题先写入公众号最终 Markdown，再调用 layout provider。
- clean HTML 必须保留正文、链接和 manifest 图片，且验证为零错误、零警告。
- `{{作者名}}`、`{{简介}}`、媒体占位和任何内部提示均为 blocker。

## 自动决策

自主模式允许总控按 workflow 的固定规则决定 topic、outline、titles、visual 和 final，但必须记录 actor=`orchestrator`、输入哈希、规则和 decision 文件。自动完成只授权生成本地交付，不授权发布。

## 回退

- 新事实或事实变化：回到 research。
- 结构变化：回到 outline。
- 标题或版本选择变化：失效 visual 及其后。
- 视觉变化：失效图片优化、发布包、layout 和 QA。
- provider 契约、路径或哈希不符：停在对应 stage，不由总控临时补造。
