# 中性核心读者画像

> 画像 ID：`core-audience`
> 版本：`1.0.0`
> 来源：用户授权的私有读者画像快照
> 用途：作为五个平台共用的第一层读者约束；运行时必须复制到 `00-intake/` 保存快照与哈希。

## 一句话画像

核心读者是 25–44 岁、集中在一线及科技产业城市的职场人。他们关心科技工具、职业成长和工作方法，希望快速理解变化、判断投入价值，并获得能在实际工作中使用的做法。

## 核心人群

### 25–34 岁职场进阶者

- 正在提升技能、效率和职业竞争力。
- 常问：新工具解决什么问题、是否值得学、怎样用于真实工作。
- 需要：通俗解释、具体步骤、真实案例、适用条件和避坑经验。

### 35–44 岁成熟职场读者

- 有较多工作经验，更关心变化对个人工作和所在行业的实际影响。
- 常问：趋势是否重要、是否值得投入时间、能否落地、代价是什么。
- 需要：判断标准、证据、边界、反例和清晰的实施方法。

## 共性特征

- 原始样本中 25–44 岁占 85%，是内容的首要服务对象；18–24 岁不是当前主力。
- 性别分布相对均衡，不采用强男性向或强女性向的默认表达。
- 读者主要分布在北京、深圳、上海、杭州等一线及科技产业城市，也包含海外读者。
- 科技数码和职场是最突出的兴趣交集，但内容应落在工作与生活中的真实问题，而非只追逐技术名词。
- 读者有一定工作经验，不需要居高临下的基础教育，也不能默认其掌握专业术语。

这些比例来自原始画像样本，只用于确定优先级，不应写成五个平台总体人群的统计事实，也不能在文章中作为外部结论引用。

## 核心任务

每篇内容至少帮助读者完成一个任务：

1. 看懂一个新概念、工具或变化。
2. 判断它与自己的工作是否相关、是否值得投入。
3. 知道下一步怎么试、怎么用，或为什么暂时不该用。
4. 识别实施成本、适用边界、失败条件和常见误区。

## 内容约束

- 先说明“这与读者有什么关系”，再展开知识背景。
- 抽象判断必须配证据、案例、操作步骤或可复核的推理。
- 给出适用对象、非适用对象、成本与限制，不把产品效果写成普遍承诺。
- 区分事实、推断、经验和建议；不虚构经历、数字、引语或结论。
- 使用准确、直接、可理解的中文。专业词首次出现时解释，但不把读者当成零基础。
- 默认提供可执行的下一步，不用空泛鼓励替代方法。

## 不属于核心画像的内容

推荐流占比、搜索占比、主页访问占比、活跃时段、发布时间和任何平台分发机制都不是中性核心画像。它们只能存在于对应平台的版本化覆盖层；目前仅小红书覆盖层保存原始画像中的此类观测。

## 运行时合并规则

读者约束按以下顺序合并，后者只能细化，不能删除前者的事实与安全边界：

1. 本文件的中性核心画像。
2. `platform-profiles.json` 中对应平台的版本化覆盖层。
3. 本篇内容明确指定的细分受众。

每个平台、每个版本的实际合并结果都必须写入本次 run 的快照，禁止只引用可变的外部文件。
