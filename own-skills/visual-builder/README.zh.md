# 视觉风格包构建

从设计图或视觉系统文档中提炼可复用规律，同时去除原作品的身份和主题。

## 什么时候用它

**从设计图构建**

我有海报、知识卡片、信息图、插画或界面，希望为一个明确的内容平台提炼可复用风格。

**编译视觉系统**

我已有结构化视觉系统说明，希望编译成带平台尺寸规则和校准图片的可移植候选包。

**批准风格包**

我已检查三张校准图和联系表，希望正式批准候选；只有我明确要求时，才继续安装到配图风格清单。

**不接**

它不处理一次性出图、普通修图、自然照片预设、Logo 提取、精确复刻，或安装未经审阅的结果。

## 它会产出什么

**生成候选不等于批准，必须先由人检查校准图片。**

- **视觉证据**：保存规范化视觉系统、来源记录、置信度和原创性检查，不保留原始图片
- **风格约定**：写入去品牌的 `style.md` 和适配平台的 `style.spec.json`
- **校准图片**：分别生成概念、流程和清单图片，并制作联系表
- **验收记录**：独立评分每张图片，选择无品牌参考图，并校验完整候选包
- **生命周期**：按草稿、等待审阅、已批准、已安装或阻断推进，不跳过门禁
- **外部动作**：使用通过核实的图片后端生成校准图，可能产生费用
- **绝不会做**：把原图交给图片生成、复制来源身份，或未经明确审阅和批准就安装

## 前置条件与边界

**前置条件**

需要可读的设计图或完整视觉系统文档、明确的目标平台和输出根目录。图片输入至少要有四类可观察设计信号，最短边不低于 512 像素。

**相邻能力分工**

| 需求 | 交给 |
|---|---|
| 提取更完整的视觉系统 | **visual-dna-system** |
| 生产文章配图 | **post-illustration-images** |

**不接的场景**

- 自然照片、纯 Logo、模糊裁片或身份元素占主导的作品
- 保留原品牌、口号、角色或精确布局的请求
- 未完成校准、验收和人工检查就要求批准或安装

**容易混淆的边界**

- 必须明确目标平台，不会根据原图尺寸猜测
- 下游安装工具不可用时，已批准的风格包仍是可移植交付物
- 文档证据不全时可以保留草稿，但不能进入等待审阅状态
