# 图片压缩

用固定命令压缩受支持的本地图片，并给出经过核实的体积变化。

## 什么时候用它

**压缩单张图片**

我有一张本地 PNG、JPEG 或 WebP，希望生成更小的 WebP 副本，同时保留原文件。

**批量压缩目录**

我需要一次处理某个本地目录中的受支持图片。只有我明确提出时，才继续扫描子目录。

**保持无损格式**

我想减小 PNG 的体积，但不能接受有损输出，希望新文件写好并核实后再汇报。

**不接**

它不处理缩放、裁剪、修图、动画、远程链接，以及 PNG、JPEG、WebP 之外的格式。

## 它会产出什么

**默认保留原图，只有得到明确授权才会替换。**

- **输出文件**：默认生成质量为 80 的 WebP，也可以按要求生成无损 PNG
- **批量结果**：同时列出成功和失败的文件，不会掩盖部分失败
- **体积报告**：返回绝对路径、压缩前后字节数和节省比例
- **结果核实**：确认每个输出文件真实存在后，才会宣布成功
- **绝不会做**：覆盖已有目标文件、下载远程图片，或改动图片内容

## 前置条件与边界

**前置条件**

需要提供一个可读的本地文件或目录。运行环境需要 Node.js 18.17 或更高版本及 `npm`；首次运行可能会把固定版本的 Sharp 安装到用户缓存。

**不接的场景**

- 同时给出多个含义不清的路径，或提供远程链接
- 动图、损坏文件及不受支持的格式
- 把压缩与裁剪、缩放或修图混在一起的请求

**容易混淆的边界**

- 是否扫描子目录必须由用户明确指定
- 自定义输出路径只适用于单个文件
- 跨格式替换会先完成并核实新文件，之后才删除原文件
