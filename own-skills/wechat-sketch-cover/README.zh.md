# 微信手绘封面

生成一张固定规格的公众号文章封面，准确处理标题，并对最终文件做确定性核验。

## 什么时候用它

**使用准确标题**

我有一个中文或中英混合文章标题，希望生成暖色手绘公众号封面，并把准确标题放在左侧。

**使用文章文件**

我有一份 Markdown 文章，希望使用开头元数据中的标题或第一个一级标题来构思封面，并准确显示文字。

**需要固定风格**

我需要标准的暖纸色、红色点缀和手绘构图，只输出一张 `1923 × 818` PNG，不想再选择视觉选项。

**不接**

它不处理其他平台、自定义风格、尺寸、配色、品牌、修图、正文配图、只写提示词、抓取链接或发布。

## 它会产出什么

**风格、布局、尺寸和文字规则全部固定，只有内容概念会随文章变化。**

- **来源记录**：在新的输出目录中保存准确标题，以及用户提供的正文或摘要
- **可复现尝试**：每次生成候选前，先保存完整提示词或构建说明
- **候选图片**：最多规范化并目视检查三次封面尝试
- **最终封面**：只有核实为 PNG 且尺寸是 `1923 × 818`，才复制为 `cover.png`
- **交付记录**：按固定顺序报告状态、标题准确性、后端、提示词、候选图和诊断
- **外部动作**：可能使用可用的图片后端或本地渲染方式制作候选
- **绝不会做**：修改原文章、覆盖已有输出，或发布封面

## 前置条件与边界

**前置条件**

需要一个含中文字符、长度为 2–35 个非空白字符的准确标题，可附文章内容；也可以提供一份可读的 Markdown 文件。环境必须有 Pillow 和内置风格资源。

**相邻能力分工**

- 一张固定公众号封面 → **wechat-sketch-cover**
- 一套文章正文配图 → **post-illustration-images**

**不接的场景**

- 多个标题来源互相冲突，且尚未选定唯一准确标题
- 不含中文字符的标题、远程链接和非 Markdown 本地文档
- 改变固定视觉约定，或加入标题之外的可读文字

**容易混淆的边界**

- 三次尝试后，只有标题准确性这一项仍有问题时，才允许交付尽力版本
- 品牌、额外文字、文件不可读或其他视觉问题都不能作为尽力版本交付
- 所有保留产物都只写入新建的输出目录
