# TranFu 网站设计规范（反推版）

> 基于当前 React/Vite 代码与 Figma 导出组件反推，覆盖页面骨架、品牌色、Logo、字体、组件、动效与内容风格。适合作为后续新增区块、重构组件、交付给设计或提示 AI 生成界面的参考。

## 1. 设计定位

TranFu 当前视觉属于 **AI Agent 系统型品牌站**，不是传统营销页。整体气质是：

- 理性、克制、工程化、可信赖。
- 以真实产品截图和工具界面作为主视觉，而不是抽象插画。
- 通过浅灰网格、命令行、状态标签、扫描/解密/流式输出等细节强化 Agent / OS / Workflow 感。
- 品牌红只做关键动作和聚焦，不大面积铺满。

关键词：`AI Agent`、`OS`、`控制台`、`工作流`、`真实产品`、`轻量科技感`。

## 2. 画布与页面结构

### 基准画布

- 设计稿宽度：`1920px`
- 页面总高：`4847px`
- 实现方式：固定 `1920px` 舞台，按 `window.innerWidth / 1920` 等比缩放，最大不超过 1。
- 页面背景：`#F0F0F0`
- 主内容宽度：核心区块多为 `1436px`，居中。
- 主内容左右常配 `240px` 浅灰侧栏作为装饰/延展。

### 页面分段

- 顶部导航：`88px` 高，固定定位。
- Hero：从 `top: 90px` 开始，高约 `988px`。
- 产品区：`top: 1080px`，高约 `1112px`。
- Skill 标题区：`top: 2194px`，高 `328px`。
- Skill 内容区：`top: 2524px`，高 `780px`。
- 实践区：`top: 3306px`，高 `820px`。
- 社区 CTA：`top: 4128px`，高 `375px`。
- 页尾：`top: 4505px`，高 `276px`。
- 域名信息：`top: 4783px`，高 `64px`。

## 4. Logo 规范

### 基础组合

- 当前主 Logo 为横向组合：左侧品牌图形标志 + 右侧 `TranFu` 英文字标。
- 标准尺寸：`135 x 32px`，其中图形标志约 `32 x 32px`，字标区域约 `88 x 21px`。
- 图形与字标应作为一个整体 lockup 使用，默认不拆分、不重新排版。
- 导航栏中 Logo 右侧可搭配 `AI Agent OS` 深色徽标，二者间距 `16px`。
- `AI Agent OS` 徽标不属于 Logo 本体，属于品牌说明 badge；在空间不足、页尾或第三方露出时可省略。

### 颜色版本

| 版本 | 颜色 | 使用场景 |
| --- | --- | --- |
| 标准红色版 | `#E63A46` | 浅灰、白色、低对比浅色背景上的默认版本 |
| 深色单色版 | `#111111` / `#2F2D2D` | 红色会与环境冲突，或需要降低品牌强调时 |
| 白色反白版 | `#FFFFFF` | 深色背景、深色图片遮罩、视频封面等场景 |

- 标准红色版优先级最高，适用于顶部导航、品牌露出、CTA 邻近区域。
- 不在红色、橙色、复杂图片或高饱和渐变背景上直接使用红色 Logo；此类场景应加浅色底板或使用白色反白版。
- 不使用透明度降低 Logo 识别度；禁用态可降低容器/按钮状态，不直接降低 Logo alpha。

### 留白与最小尺寸

- 安全留白以图形标志高度 `X` 为基准：Logo 四周至少保留 `0.5X` 空白。
- 与导航链接、按钮、语言切换器等交互元素的最小间距不低于 `32px`；与 `AI Agent OS` badge 间距固定 `16px`。
- 桌面端主导航 Logo 高度保持 `32px`。
- 移动端或紧凑场景中，横向 Logo 高度不低于 `24px`；低于该尺寸时优先只使用图形标志。
- 单独图形标志最小显示尺寸不低于 `20 x 20px`，低于该尺寸不作为可识别品牌标识使用。

### 使用方式

- 顶部导航使用「Logo lockup + `AI Agent OS` badge」作为标准组合。
- 页尾、社交头像、favicon、App icon 等空间受限场景可只使用图形标志。
- Hero 或标题区如需品牌露出，优先使用文字内容中的 `TranFu`，避免重复堆叠完整 Logo。
- 产品截图内若已出现 Logo，页面外层不再额外添加装饰性 Logo 水印。
- Logo 附近避免同时出现多个红色装饰元素，确保品牌红的视觉焦点明确。

### 禁用规则

- 不拉伸、压扁、旋转、倾斜或重新绘制 Logo。
- 不改变图形与字标的比例、间距或相对位置。
- 不给 Logo 添加投影、描边、渐变、发光、纹理或 3D 效果。
- 不把 Logo 放进胶囊按钮、标签或复杂卡片装饰中作为普通图标使用。
- 不用 Logo 替代普通功能图标；功能按钮优先使用明确的 UI 图标。

## 5. 字体与排版

### 字体族

- 中文和常规 UI：`MiSans`
- Hero 大标题：`Alimama_ShuHeiTi`，`font-weight: 700`
- 英文徽标 / 命令行：`Hammersmith One`
- 回退：`sans-serif`

### 字号层级

| 层级 | 字号 | 字重 | 行高 | 用途 |
| --- | --- | --- | --- | --- |
| Hero H1 | `92px` | 700 | `1.02` | 首屏主宣言 |
| CTA H1 | `64px` | 600 | normal | 社区召唤区标题 |
| Section H2 | `56px` | 600 | normal | 产品、Skill、实践区标题 |
| Card Title | `20px` / `24px` | 600 | normal | Skill 卡片、产品标题 |
| Body | `16px` | 400 | `1.8` | 导航、正文、说明 |
| Meta | `14px` | 400/500 | normal / `1.8` | 标签、按钮、页尾 |
| Tag | `12px` / `11px` | 400 | normal | 小标签、计数、阅读时间 |

### 排版规则

- 中文长说明使用 `line-height: 1.8`，制造舒展但不松散的阅读节奏。
- 大标题居中，正文和卡片内容多为左对齐。
- 不使用负字距；保持默认 letter spacing。
- 大标题避免超过两行，Hero 可通过焦点动效突出关键词。

## 8. 组件规范

### 顶部导航

- 高度：`88px`
- 背景：`#F7F7F7`
- 底部分割线：`1px #E6E6E6`
- 内容宽：`1440px`，左右居中。
- Logo 使用品牌红，旁边可带深色 `AI Agent OS` 徽标。
- 导航文字：`16px MiSans #333`，hover 变 `#E63A46`。
- 主按钮：`98 x 40px`，红底白字，圆角 `12px`。

### 主按钮

- 背景：`#E63A46`
- Hover：`#D02E3A`
- 文字：白色，`16px` 或 `18px`
- 高度：常用 `48px`，导航内为 `40px`，小卡片内为 `32px`
- 圆角：`12px` 或 `16px`
- 动效：hover 轻微上移 `1-2px`，tap 缩放约 `0.96`

### 分段控件 / Tabs

- 外壳背景：`#F0F0F0` 或 `#E6E6E6`
- 外壳 padding：`4px`
- 外壳圆角：`16px`
- 选中项：白底，圆角 `12px`
- 产品 Tab：`240 x 56px`
- Hero Tab：`108 x 40px`
- 文字：`14-16px`，选中 `font-medium`

### 标签 / Badge

- 状态标签：`32px` 高，`px 16px`，圆角 `8px`
- 小属性标签：`24px` 高，`px 8px`，圆角 `6px`，描边 `#E6E6E6`
- 最新/热门：橙色文字 `#FA5D19`，浅橙底 `rgba(250,93,25,0.1)`
- Skill/工具标签：灰字 `#999`，白底或浅灰底。

### 卡片

- Skill 卡片：白底、圆角 `16px`、默认无阴影。
- 卡片 hover：`0 18px 48px rgba(17,17,17,0.12-0.16)`
- 产品卡片不是传统卡片，而是在大浅灰容器中使用信息区 + 截图区。
- 卡片内部常用 `16px` padding，标题/描述/标签之间用 `16-24px` 间距。

### 截图/媒体

- 产品展示必须使用真实产品截图。
- Hero 演示截图采用暗色遮罩 `rgba(0,0,0,0.4)`，外层半透明边框 `rgba(17,17,17,0.1)`。
- 产品截图容器圆角 `30px`，图片 `object-cover`。

## 9. 装饰语言

### 背景网格

- 使用 `120 x 120px` 浅灰方块，`2px` gap，圆角 `12px`。
- 基础色 `#F7F7F7`，少量变化块 `#F2F2F2`。
- 左右镜像布局，保持系统面板感。

### 数据/Agent 感元素

- 灰色短线模拟流式输出：高 `4px`、圆角 full、色 `#D9D9D9`。
- 方括号水印：如 `[ Task Type ]`、`[ Json ]`、`[ Model ]`，`14px #CCC`。
- 品牌红星形/焦点元素可轻量出现。
- 在线状态点：外圈 `#35C759` 10% 透明，内点实色。

### 纹理

- 可使用低透明度 canvas 纹理作为背景，通常 `opacity: 0.3`。
- 纹理只做气氛，不影响文字可读性。

## 10. 动效规范

### 运动曲线

- 常用 ease：`[0.22, 1, 0.36, 1]`
- 常用入场：`opacity 0 -> 1` + `y 12-24px -> 0`
- 入场时长：`0.55-0.8s`
- Tab 内容切换：`0.3-0.5s`
- Hover：轻微位移或缩放，不做夸张弹跳。

### 特色动效

- Hero 标题使用 TrueFocus 聚焦框，红色边框，低模糊。
- Badge 文案使用解密/打字动效。
- 背景装饰可慢速漂浮：`7-9s` 循环。
- 流式输出短线循环显示，强化 Agent 正在工作。

### 可访问性

- 已支持 `prefers-reduced-motion: reduce`：动画时长收敛到极短。
- 后续新增动效也应尊重 reduced motion。

## 11. 内容与文案语气

### 语气

- 简洁、具体、工程化。
- 多使用“真实工作”“复制即用”“实测”“驾驶舱”“看板”“路径”“闭环”等可执行词。
- 避免空泛营销词堆叠。

### 命名模式

- 产品：`TranFuBar`、`TranFu//Agents`、`GitHub Learning Lab`
- 功能标题：`代码库理解驾驶舱`、`30 日抢救陪练`、`团队级 Agent 实时看板`
- CTA：`开始使用`、`加入我们`、`加入社群`、`安装 Skill`

## 12. 新增页面/组件时的设计检查清单

- 是否使用 `#F0F0F0` 页面背景和 `#F7F7F7` 大区块底板？
- 品牌红是否只用于主动作、品牌和焦点？
- Logo 是否使用标准 lockup、正确颜色版本、足够留白和最小尺寸？
- 主内容是否保持 `1436px` 居中或沿用现有 1920 舞台？
- 字体是否优先使用 `MiSans`，大标题是否符合现有层级？
- 圆角是否落在 `6/8/12/16/24/30/40` 体系内？
- 卡片是否默认轻、hover 再出现阴影？
- 是否使用真实产品图或界面截图，而不是抽象装饰？
- 动效是否轻、慢、克制，并支持 reduced motion？
- 文案是否明确说明“解决什么真实问题”？
