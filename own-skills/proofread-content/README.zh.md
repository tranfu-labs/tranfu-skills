# proofread-content

把完整简体中文成稿变得自然、干净，同时守住事实、立场、结构和作者声口。

## 什么时候用它

**审校粘贴成稿**

我在对话中提供了一篇完整自媒体文章，希望经过三轮审校后再进入排版。

**审校本地文件**

我有一个可读的 UTF-8 编码 `.md` 或 `.txt` 文件，并允许在全部保真检查通过后原地更新。

**保留作者语气**

我想减少重复或模板化 AI 表达，但不希望凭空增加经历、把全文改成聊天口吻，或破坏适合当前平台的声口。

**不接**

从零写作、翻译、排版、发布、爆款重构、专业法律或学术文本，以及规避 AI 检测，都不属于本流程。

## 它会产出什么

**全部保真检查通过之前，绝不会写回正文。**

- **三轮审校**：依次检查逻辑与内部一致性、结合语境检查二十四类 AI 表达，再修正语言细节和阅读节奏。
- **纯净正文**：文件输入通过原子替换原地更新；粘贴输入在当前工作目录创建或覆盖 `proofread-content.md`。
- **结构化结果**：对话中只返回一个 `PROOFREAD_RESULT`，列出状态、平台、输出路径、门禁结果和三轮摘要。
- **受保护内容**：标题层级、事实、数字、日期、金额、版本、专有名词、引用、链接、案例和 Markdown 语义保持不变。
- **失败保护**：状态为 `NEEDS_AUTHOR_INPUT` 或 `BLOCKED` 时，现有目标文件逐字节不变，并清理临时文件。
- **绝不会做**：不创建备份，不增加事实或行动号召，不默认核验外部事实，不发布，也不承诺绕过检测器。

## 前置条件与边界

**前置条件**

必须提供一篇完整简体中文成稿，可以直接粘贴，也可以给出一个非空的 UTF-8 编码 `.md` 或 `.txt` 文件。内置去模板化表达规则和平台语体说明必须可读。

**支持平台**

支持微信公众号、小红书、微博、知乎、头条号，并可在无法判断时使用通用自媒体语体。用户明确指定其他平台时会停止处理。

**写入边界**

- 文件输入只授权在检查通过后原子替换该文件。
- 粘贴输入只授权在当前目录创建或覆盖 `proofread-content.md`。
- 原文存在事实冲突、未完成占位或必须由作者决定的信息时，会停止写入并提出具体问题。

**微妙边界**

`READY_FOR_LAYOUT` 只表示文章通过本流程的编辑检查，不代表外部事实、法律合规或平台最新政策已经核验。
