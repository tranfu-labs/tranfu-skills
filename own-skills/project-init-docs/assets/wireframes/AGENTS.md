# 线框图（字符图）· 工作约定

这个目录放**字符图线框**——用等宽字符快速对齐每个页面"有哪些信息块、整体框架、谁在什么位置"。
它只表达**信息架构与版式**，不表达视觉样式（颜色 / 圆角 / 控件状态变体），那些属于 UX/视觉层，字符图不碰。

本目录在初始化时**默认生成**。若本项目确定为无界面的工具 / 库 / CLI / SDK 类，按仓库根 `AGENTS.md` 的「线框图」一节删除整个 `docs/wireframes/`。

页面来自真实路由，不凭空造。新增页面：复制 `_template/page.md` 到 `pages/<页面名>.md` 再填。

## 比例尺（硬规则）

单位是**显示列**（等宽字体里实际占的列数），不是字符个数（codepoint）：

- 横向：1 显示列 = 12px。**全角字符（中日韩，东亚宽度 W/F）占 2 列；其余一律占 1 列**——半角、框线、以及歧义宽字符（圆点编号 `①②`、`◯ · —` 等，东亚宽度 A）统一按 1 列计。
- 纵向：1 行 = 24px（全角字高约为半角宽的 2 倍，横 12 / 纵 24 让格子长宽比贴近真实像素）。

常用断点换算（真实 px → 显示列）：

| 断点 | 真实尺寸 | 显示列 × 行 |
|---|---|---|
| 桌面 | 1440×900 | 120 列 × 38 行 |
| 平板 | 768×1024 | 64 列 × 43 行 |
| 手机 | 375×812 | 31 列 × 34 行 |

每行必须补齐到右边框正好落在目标列（桌面第 120 列），框线才真对齐——这是字符图"等宽对齐"价值的前提。一行纯中文 = 60 个字 × 2 列 = 120 列。行数（38/43/34）是版面上限，不强制填满。多个断点共用这把尺，画在一起时盒宽天然成比例（手机框约为桌面的 1/4）。

## 尺寸头（硬规则）

尺寸两层都要写，便于核对：

1. **页首声明块**：比例尺 + 本页覆盖的断点清单（含真实 px）。
2. **每个断点框上方一条尺寸条**：`真实px（显示列 × 行）`，其中显示列数必须 = 真实px ÷ 比例尺（桌面 120 / 平板 64 / 手机 31），对不上即不合格。

校验显示列宽**唯一推荐**用 Python 按东亚宽度计列（与上面口径一致）：

```
python3 -c "import sys,unicodedata as u; [print(sum(2 if u.east_asian_width(c) in ('W','F') else 1 for c in l.rstrip('\n')), l.rstrip('\n')) for l in open(sys.argv[1])]" 文件.md
```

**禁用：** `awk '{print length}'`（中文+框线下既非字节也非列）、codepoint 计数（把全角错记成 1 列）、以及 `wc -L`（macOS/BSD 下按字节或返回 0，不可靠；仅 GNU coreutils + UTF-8 locale 下才近似正确）。

口径自洽即可：脚本与画图都按"全角 2 列、歧义/半角 1 列"，则每行严格落在目标列。若在某些 CJK 终端预览时框线看起来整体右凸，是该终端把歧义宽字符渲染成 2 列所致，非字符图本身错误。

## 字符集与控件画法

- 实线边框：`┌ ┐ └ ┘ ├ ┤ ┬ ┴ ┼ ─ │`
- 浮层用双线框：`╔ ╗ ╚ ╝ ═ ║`
- 占位符：文字块 `≈≈≈`；图片/媒体 `▢` 或 `[图]`；输入 `[____]`；下拉 `[ 文字 ▾ ]`；按钮 `[ 文字 ]`；复选/单选 `☐ ◯`；开关 `( ● )`

控件的统一画法见 [legend.md](legend.md)，画页面时一律照它，不自创画法、不画状态变体。

## 消歧义（硬规则）

字符形状会撞车（标签页 / 表格一行 / 一排按钮长得一样），所以容器必须标明身份：

- **盒子类容器**（卡片 / 表格 / 弹窗 / 页面区块）：在**顶边左角嵌中文类型标签**，如 `┌─ 卡片 ──┐`、`┌─ 表格 ──┬──┐`、`╔═ 弹窗 ═╗`。
- **线性控件**（标签页 / 面包屑 / 工具栏）：行首加**中文类型前缀**，如 `标签页 ‹标签A› 标签B`。
- 标签一律用中文。简单控件（按钮 / 输入 / 下拉）字符本身已不歧义，不强制加标签。

## 注释

- 框内元素打编号圆点 `①②③…`，同一元素在多个断点共用同一编号。
- 注释正文用紧跟字符图后的表格承载，列固定：`编号 | 元素 | 状态/交互 | 数据来源 | 引用控件`。
- 编号与注释表必须一一对应，不留孤儿编号。

## 流转图（flow.md）

`flow.md` 是项目级、一份就够，画**页面之间怎么走**，与单页 `page.md`（页面内部长什么样）正交。它是流程图、不是视口，**不套比例尺、不分断点**。规则：

- **按用户流程分节**：`## 登录流程`、`## 忘记密码流程`、`## 新建任务流程`…，每节一张字符流程图 + 一张步骤表。
- **节点 = 真实页面**：实线框照消歧义规则标身份 `┌─ 页面名 路由 ─┐`，框内一行指向该页 `pages/<page>.md`。同一页面可在多个流程里复现。
- **同页态变化**（无跳转，如提交后刷新、提示态）：用**虚线框 `┄ ┊` 节点**区分，不伪造成新页面。
- **边 = 跳转**：箭头 `─▶ ◀─ ▼ ▲ │`，每条边打编号；步骤表列固定 `步 | 从 | 到 | 触发`，与图中编号一一对应、无孤儿编号。
- 入口流程（如登录）放前面；流程间能衔接的写「→转 XX流程」。
- 流程来自真实路由 + 产品意图归纳；推不出的标 `TODO: 需人工确认`，不硬造。
