# format-content

把一篇完整 Markdown 文章转换成可复制到公众号的 HTML，同时保持原文含义不变。

## 什么时候用它

**排版粘贴文章**

我在对话中提供了一篇结构完整的 Markdown，希望套用固定红白主题完成公众号排版。

**排版本地文件**

我有且只有一个可读的 `.md` 文件，需要一份纯净的行内样式片段和一份带复制按钮的浏览器预览。

**不接**

Word、PDF、TXT、富文本、无结构散文、其他主题、普通网页、实质性改写、创建公众号草稿或调用公众号接口发布，都不属于本流程。

## 它会产出什么

**纯净片段必须达到零错误、零警告，才会生成预览。**

- **纯净文件**：生成 `{stem}_排版_红白色系(red-white).html`，内容只有一个外层 `<section>`，全部使用行内样式。
- **预览文件**：生成 `{stem}_排版_红白色系(red-white)_预览.html`，原样嵌入已校验片段，并提供复制按钮。
- **内容保真**：保留段落、列表、表格、代码、媒体、链接、标题和事实含义，只有文章一级标题可以不进入正文。
- **固定主题**：只使用内置红白主题与公共组件，不另选或临时发明视觉风格。
- **反复校验**：运行内置校验脚本，清除全部 `ERROR` 和 `WARNING` 后再检查预览。
- **绝不会做**：不调用公众号接口，不创建草稿，不发布，不改写事实，也不会把结果写到源文件旁边。

## 前置条件与边界

**前置条件**

必须提供唯一且非空的 Markdown 来源，可以是粘贴内容或一个可读的 `.md` 文件。内置主题说明、校验脚本、预览封装脚本和预览模板也必须齐全。

**输出位置**

两份文件始终写入运行时当前工作目录。即使源文件在其他目录，输出位置也不会随之改变。

**输入范围**

- 支持：来源明确、结构完整的一篇 Markdown 文章。
- 不支持：`.doc`、`.docx`、`.pdf`、`.txt`、HTML、富文本、多个含糊来源，或需要临时猜测 Markdown 结构的散文。
- 作者或媒体信息缺失时会保留明确占位或删除对应空行，绝不虚构。

**微妙边界**

排版可以选择组件并调整阅读节奏，但不得添加事实、图片说明、链接、作者、引导语或其他实质内容。
