# 五平台选题适配手册

本手册用于把同一个公共主题转译为五个平台各自成立的消费理由。不是把一个标题机械改写五次，也不是生成标题池。

默认平台与顺序：

1. 微信公众号
2. 小红书
3. 知乎
4. 微博
5. 今日头条

用户明确指定子集时只输出指定平台，但主选题仍需在五个平台都没有明显弱适配。

## 通用规则

### 共享内核，分别包装

五个平台必须共享：

- 同一个可核验事实基础。
- 同一个公共核心角度。
- 同一个核心目标人群及其问题。
- 不冲突的判断与行动结论。

五个平台可以不同：

- 切入场景与开头钩子。
- 标题原型的句式和关键词。
- 信息密度、论证深度和互动设计。
- 内容形式与材料排列顺序。

如果必须改变核心事实或结论才能适配某个平台，说明它不是同一选题的适配版本，应降低跨平台适配评级。

### 标题原型

每个平台只给一个标题原型。原型必须表达该平台为什么值得消费这条内容，并满足以下约束：

- 主体、动作、数字、时间和结果有证据支撑。
- 不虚构亲历、读者反馈、平台排名或社会共识。
- 不用空泛的“震惊”“炸裂”“所有人都在用”替代具体价值。
- 不把问号当成规避事实责任的工具。
- 长度建议只作为本 Skill 的内部工作约束，不宣称为平台当前的官方字符上限，也不得凭模型记忆改写成官方规则。

### 适配评级

- `strong`：主题天然符合平台消费动机，有原生内容形式和清晰互动/传播理由。
- `usable`：经过合理切角可以成立，但不是该主题最强的平台。
- `weak`：只能硬套标题或删减内容，缺少平台原生价值。

主选题五个平台都必须是 `strong` 或 `usable`。不要在最终产物显示这些内部评级；将其翻译成一句具体的“适配理由”。

### 统一画像的使用

内置画像中的首页推荐、搜索、个人主页和活跃时段比例，原始语境来自小红书，但用户已授权作为跨平台统一假设。它们用于理解这个固定账号的读者，不得写成其他四个平台的官方流量规律。

### GOOD：同一主题的五平台转译

以下方括号内容均来自同一批已核验证据，不代表真实事件：

- 公共事实：`[主体发布了已核验能力 E]`
- 核心角度：`[E 改变的是知识工作的任务分工，不等于整体岗位替代]`
- 目标人群：`[需要判断 E 是否影响当前任务的知识工作者]`

| 平台 | 标题原型 | 适配理由 | 内容形式 |
|---|---|---|---|
| 微信公众号 | `[E 真正改变的，不是工具，而是知识工作的分工]` | 完整解释机制、证据和职业影响，形成信任与转发价值 | 深度解读 |
| 小红书 | `[岗位]先别急着用 E：先做使用检查` | 落到使用场景并形成可收藏、可搜索的检查表 | 清单图文 |
| 知乎 | `E 是否真的会替代[岗位任务]？` | 问题存在分歧，可用证据、反例和边界回答 | 深度问答 |
| 微博 | `E 刚发布：先别把“能做”误读成“能替代”` | 抓住讨论窗口并提供有边界的立场 | 事件速评 |
| 今日头条 | `[主体]发布 E，哪些职场任务先受影响？` | 主体、动作和大众影响清楚，具备新闻入口 | 新闻解读 |

WRONG: 五个平台都写“E 来了”，适配理由都写“平台用户关注热点”，内容形式都写“图文”。

Reason: 五行没有形成不同的平台消费理由，也没有说明标题承诺怎样由同一个事实与角度兑现。

## 微信公众号

### 核心消费理由

深度、信任与转发。读者愿意投入较长时间，是为了看懂变化、形成判断，或把一篇足够完整的文章转给同事和朋友。

### 强适配信号

- 主题需要背景、机制、影响和行动建议的完整解释。
- 有多条证据、案例或对比，能建立可信判断。
- 对团队、行业或职业选择有讨论价值，值得转发。
- 能从新闻事件上升为可复用框架，而不是只追逐当天情绪。

### 弱适配信号

- 只有一句立场或一张截图，没有展开空间。
- 只靠短时热梗成立，隔天即失去信息价值。
- 主题需要现场互动，却缺少可沉淀的判断。

### 标题原型

- 推荐工作长度约 20–28 个中文字符，优先清楚而不是塞满修饰词。
- 常用结构：`变化/事件 + 对知识工作者的真实影响`、`问题 + 判断框架`、`现象 + 为什么现在值得关注`。
- 可以保留一层悬念，但要让读者知道文章讨论的对象和价值。
- 避免只有工具名、夸张情绪或“深度长文”自我标榜。

### 内容形式

- 深度解读
- 趋势判断 + 行动框架
- 案例拆解 + 方法清单
- 多来源事实梳理
- 工具/工作流评测框架

### 起草提示

- 开头重点：用一个变化与读者工作之间的冲突建立必要性，不先铺长背景。
- 展开重点：先事实，再解释，再影响，最后给判断标准或行动方法。
- 形式提示：适合分节、证据引用、对比表、清单和可转发总结。

### 适配理由写法

说明为什么该主题值得完整解释、为什么能建立信任、读者为何愿意转发。

WRONG: `适合长文。` Reason: 只描述篇幅，没有说明主题为何需要完整论证、能建立信任或值得转发。

## 小红书

### 核心消费理由

场景、收藏与搜索。读者要快速判断“这与我的工作有什么关系”，并得到可以保存、复用或搜索回来的方法。

### 强适配信号

- 能落到具体岗位、任务、工具选择或工作场景。
- 可以形成步骤、检查表、避坑清单、对比或模板。
- 有明确搜索词，例如工具名、功能名、岗位问题或方法名。
- 即使读者从首页第一次看到账号，也能独立理解价值。

### 弱适配信号

- 只有宏大趋势，没有个人场景或可保存信息。
- 依赖复杂前史和长篇论证才能成立。
- 只是把新闻摘要拆成卡片，没有行动价值。

### 标题原型

- 推荐工作长度约 12–20 个中文字符，优先主体前置、场景明确。
- 常用结构：`人群/场景 + 问题或收益`、`工具/变化 + 实测前检查项`、`别急着做 X，先看 Y`。
- 可以使用数字，但数字必须对应真实步骤、误区或对比项。
- 保留工具名、事件名和核心问题词，兼顾搜索；不堆无关热词。

### 内容形式

- 清单型图文笔记
- 步骤教程或工作流
- 对比卡片
- 避坑笔记
- 场景案例 + 可执行模板

### 起草提示

- 开头重点：先报具体工作场景和结果预期，迅速完成“与我有关”的判断。
- 展开重点：用短段落、步骤、对比和注意事项给出可收藏信息。
- 形式提示：适合 6–9 页图文卡片或结构清楚的笔记，关键词自然出现，不做标签堆砌。

### 适配理由写法

指出具体场景、可收藏资产和搜索需求。

WRONG: `年轻人多，适合短内容。` Reason: 用人群刻板印象代替了场景、收藏资产和搜索需求。

## 知乎

### 核心消费理由

问题价值、证据与论证。读者进入内容是为了得到一个问题的可靠答案，理解不同观点为什么成立或不成立。

### 强适配信号

- 能表述为一个真实、具体、可回答的问题。
- 存在认知分歧、决策难点或常见误解。
- 有足够证据解释机制、适用条件、反例和边界。
- 回答能沉淀为长期可检索的知识，而不只是实时播报。

### 弱适配信号

- 只有情绪立场，没有需要论证的问题。
- 核心信息一句话即可说完，没有解释空间。
- 依赖未验证传闻，无法处理反方观点。

### 标题原型

- 优先使用自然问题句或明确判断题，工作长度约 18–32 个中文字符。
- 常用结构：`X 是否意味着 Y？`、`如何判断 X 值不值得……？`、`为什么 X 发生了，但 Y 未必成立？`
- 问题中应包含对象、条件或受影响人群，避免“如何评价”式空泛提问。
- 不在问题中预埋无法证明的结论。

### 内容形式

- 深度问答
- 概念澄清 + 证据论证
- 正反观点比较
- 决策框架
- 机制解释 + 案例验证

### 起草提示

- 开头重点：先给短答案和判断范围，再说明为什么问题容易被误解。
- 展开重点：定义概念、列证据、解释机制、处理反例和适用边界。
- 形式提示：适合引用原始资料、列出判断条件和回应潜在反驳。

### 适配理由写法

说明该主题解决什么真实问题、有哪些证据和分歧值得论证。

WRONG: `知乎用户理性。` Reason: 没有指出真实问题、证据基础或值得处理的观点分歧。

## 微博

### 核心消费理由

时效、立场与讨论。读者需要快速知道发生了什么、这件事怎样理解，以及自己是否愿意参与表达。

### 强适配信号

- 事件正在发生，72 小时内有明确公开关注信号。
- 能在一两句话内给出有边界的鲜明判断。
- 存在可讨论的利弊、影响对象或行动选择。
- 有可以直接引用或链接的原始事实，不依赖长背景才能理解。

### 弱适配信号

- 主题完全依赖长篇教程或复杂论证。
- 没有当下讨论入口，只有常青知识总结。
- 为制造讨论而极化立场或挑动焦虑。

### 标题原型

微博通常没有独立文章标题时，将“标题原型”视为首句钩子：

- 推荐首句约 15–30 个中文字符，先说事件或判断，不写空洞导语。
- 常用结构：`事件 + 我的边界判断`、`别把 X 误读成 Y`、`真正受影响的不是 A，而是 B`。
- 可以带讨论问题，但问题必须建立在已核验事实之上。
- 不伪造热搜排名、全网态度或“大家都在……”的共识。

### 内容形式

- 单条观点 + 证据链接
- 多条串联的微博线程
- 事件速评
- 对比图或信息卡
- 投票/提问式讨论（选项必须中性）

### 起草提示

- 开头重点：事件与判断同屏出现，让读者立即知道立场边界。
- 展开重点：用最少背景支撑判断，再给一个与职场人相关的讨论问题。
- 形式提示：重要事实可拆成线程；避免把复杂条件压缩成绝对结论。

### 适配理由写法

说明当前讨论窗口、可表达的边界立场和互动问题。

WRONG: `适合蹭热搜。` Reason: 只追逐注意力，没有说明可核验事件、边界立场或真实讨论价值。

## 今日头条

### 核心消费理由

新闻价值、大众相关性与信息清晰度。读者要迅速理解事件主体、发生了什么、影响谁，以及接下来应关注什么。

### 强适配信号

- 有明确事件、主体、时间和可核验进展。
- 影响范围不局限于极小技术圈，能解释普通职场人的关联。
- 材料足以按事实—影响—建议的顺序清晰展开。
- 有新闻入口，同时提供比快讯更多的背景或实用信息。

### 弱适配信号

- 只有抽象概念或个人感受，没有事件和信息增量。
- 标题依靠悬念，正文却无法说明主体和影响。
- 只面向极窄专业人群，无法解释大众相关性。

### 标题原型

- 推荐工作长度约 20–30 个中文字符，主体、动作和影响尽量完整。
- 常用结构：`主体 + 新动作 + 对某类人意味着什么`、`事件进展 + 影响解释`、`新变化落地，哪些人先受影响`。
- 新闻性不等于夸张；避免“重磅”“突发”滥用，以及没有依据的巨大影响。
- 数字和时间必须来自可核验来源，正文能够完整兑现。

### 内容形式

- 新闻解读
- 事件时间线 + 影响说明
- 大众科普
- 政策/产品变化梳理
- 事实问答 + 行动建议

### 起草提示

- 开头重点：先交代主体、动作、时间和影响对象，再补背景。
- 展开重点：按事实、原因/机制、影响范围、读者行动排列信息。
- 形式提示：使用短小标题、时间线、要点和来源，降低理解成本。

### 适配理由写法

说明新闻入口、大众影响和信息增量。

WRONG: `头条用户喜欢新闻。` Reason: 没有指出具体新闻入口、影响对象或相对快讯的信息增量。

## 跨平台自检

为主选题生成平台表前按以下过程检查。复用父 Skill 的现有待办，不创建第二份；若本段被独立执行，CREATE A TODO LIST FOR THE TASKS BELOW，并把以下六步各建为一个 TODO：

1. 校验五个平台是否共享同一事实、核心角度和目标人群。若核心事实或结论不一致 → 停止适配，把相应候选标为 `weak`，返回候选筛选。
2. 校验五个标题原型能否由同一证据基础兑现。若任一标题改变事实边界 → 修正标题并重新执行步骤 2；无法修正 → 标为 `weak` 并返回候选筛选。
3. 校验每个平台是否有不同且具体的适配理由，以及符合消费动机的内容形式：公众号深度、小红书收藏、知乎论证、微博讨论、头条清晰新闻。若只是换平台名或形式不符 → 修正后重新执行步骤 3；无法修正 → 标为 `weak` 并返回候选筛选。
4. 校验五个平台均为 `strong` 或 `usable`。若任一仍为 `weak` → 取消主选题资格并返回候选筛选。
5. 确定 `render_platforms`：用户明确指定子集 → 设为该子集；否则 → 设为全部五个平台。
6. 步骤 1–4 全部通过后，按 `render_platforms` 渲染一次平台适配表并结束。
