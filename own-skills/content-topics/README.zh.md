# content-topics

在正式写稿前选出站得住的方向，同时配好证据、平台版本和可执行简纲。

## 什么时候用它

**从材料做选题**

我有调研目录、事实库或一批来源材料，想从中收敛出一个主选题和若干可信备选。

**寻找热点选题**

我需要给固定账号找近期角度，希望同时核验事实依据和公众关注信号，而不是把热度当成事实。

**适配多个平台**

我想把同一个内容机会分别适配微信公众号、小红书、知乎、微博和今日头条，也可以明确只要其中几个平台。

**不接**

主题已经确定、只需要标题、全文、正文改写、资料搜集、内容日历、发布或表现分析时，应交给其他流程。

## 它会产出什么

**它完成选题决策，然后停在全文写作之前。**

- **选题结论**：证据充分时给出一个主选题和四个备选。
- **证据边界**：记录来源、时效、可信度和未解决缺口，不编造热度数据。
- **平台版本**：每个平台都有标题原型、适配理由和内容形式，主选题另有共享简纲。
- **文件写入**：独立模式新建 `02-选题方案.md`，已有同名文件时增加时间戳，绝不覆盖旧方案。
- **总控模式**：只写约定的调研发现文件，并把结果交回总控流程。
- **绝不会做**：不修改来源材料，不保存临时画像，不写完整文章，不发布内容，也不展示内部评分。

## 前置条件与边界

**前置条件**

内置账号画像、选题规则和平台手册必须可读。可以提供本地材料、粘贴证据或公开链接，也可以允许检索近期热点；没有材料时必须具备实时检索能力。

**证据要求**

- 时效型主选题必须同时有可信事实来源，以及最近七十二小时内可追溯的关注信号。
- 证据不足时结果为 `NEEDS_EVIDENCE`；必需输入不可读或无法实时检索时结果为 `BLOCKED`。
- 遇到登录、验证码或付费墙时，不会根据上下文猜测页面内容。

**相邻流程分工**

| 需要做的事 | 交给 |
|---|---|
| 生成或优化标题 | `title-options` |
| 撰写完整文章 | 正文写作流程 |
| 搜集并维护来源 | 调研流程 |

**微妙边界**

为说明选题包装而给出的单个标题原型属于本流程；为已完成文章批量生成标题，则属于标题生成任务。
