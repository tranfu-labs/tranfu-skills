# AI 创业方向禁止清单 v1.0

> 13 条结构性禁止方向，按风险等级分为 Tier 1（绝对禁止）/ Tier 2（高度警惕）/ Tier 3（谨慎进入）。
> 每条均提供：典型形态、禁止理由（三维评分）、死亡案例、唯一豁免条件。

---

## Tier 1 · 绝对禁止（三维全负，结构性必死）

### 1.1 单功能 AI 工具

**典型形态**：AI 写作、AI 翻译、AI 画图、AI 视频剪辑、AI PPT 生成、AI 简历优化、AI 邮件助手。共同特征：产品 = 一个模型能力 + 一个轻薄 UI。

**三维评分**：替代风险 -2 / 杠杆方向 -2 / 单位经济 -2

**禁止理由**：
- 每次基础模型升级覆盖 80% 单点工具核心能力（ChatGPT Canvas、Claude Artifacts 已直接吸纳大半 PPT / 写作工具）
- 模型越强，用户越倾向直接使用 ChatGPT / Claude 原生界面
- CAC 持续上涨（2024-2026 涨 3-5 倍），LTV 因用户随时切换而受限

**死亡名单**：Jasper.ai（估值从 15 亿跌至大规模裁员）、Copy.ai（增长停滞）、MagicSlides 等数十家 AI PPT 工具

**唯一豁免条件**（须满足至少一项）：
- 拥有该垂直场景下模型训练数据无法覆盖的独家数据集
- 已建立有真实留存的创作者社区或品牌（Midjourney 模式）
- 已有自有分发渠道（Cursor 在开发者社区、Notion AI 在已有用户中）

---

### 1.2 通用 Agent / Agent 操作系统

**典型形态**："让一个 Agent 帮你处理所有事"、"AI 个人助理"、"通用任务自动化框架"、"AI Copilot 全场景"。

**三维评分**：替代风险 -2 / 杠杆方向 -2 / 单位经济 -1

**禁止理由**：
- 这正是 Anthropic Claude Agent、OpenAI Operator、Google Project Mariner 的 roadmap 第一优先级
- 模型能力越强 = 通用 Agent 越应该由模型公司直接提供
- 操作系统级别产品全球只能存活 2-3 家——它们一定是模型公司本身

**唯一豁免条件**：无。任何挑战"操作系统位"的早期公司都没有生存机会。

---

### 1.3 通用浏览器 / 桌面 Agent（Computer Use 对标）

**典型形态**：让 AI 操作浏览器、点击网页、控制 Mac / Windows 桌面、自动化完成跨应用任务的"通用"产品。

**三维评分**：替代风险 -2 / 杠杆方向 -2 / 单位经济 -1

**禁止理由**：
- Anthropic Computer Use 已 GA、OpenAI Operator 已发布、Adept 已被 Amazon 收购，该赛道 2024-2025 死亡率超 60%
- Computer Use 已是模型公司的官方原语，第三方做"通用版"无差异化空间
- 幸存者（Browserbase、Browser Use）均已降维做"给开发者的浏览器基础设施"

**唯一豁免条件**：不做"通用浏览器 Agent"，改做"给开发者的浏览器基础设施"。这本质上是另一个方向。

---

### 1.4 通用 AI 搜索

**典型形态**：对标 Perplexity 做中文版、AI 学术搜索、AI 实时搜索等通用 RAG + 模型整合的搜索产品。

**三维评分**：替代风险 -2 / 杠杆方向 -2 / 单位经济 -2

**禁止理由**：
- 搜索是模型公司核心战略，ChatGPT Search / Claude Web Search / Google AI Overview / 字节豆包 / 腾讯元宝 / Kimi 探索全部在卷
- 搜索边际成本随调用线性增长，且需持续支付 Bing / Google API 费用
- Perplexity 估值 90 亿美金但 ARR 不到 1 亿，且持续亏损

**死亡名单**：You.com（裁员 50%+ 转型 B 端 API）、Andi（关停）

**唯一豁免条件**：无新进入者机会。

---

## Tier 2 · 高度警惕（两维负面，存活率 < 15%）

### 2.1 AI Agent / 技能交易市场（Marketplace）

**典型形态**：Agent 商店、AI 技能交易平台、双边能力市场、"人人都能上架 AI 能力"。

**三维评分**：替代风险 -1 / 杠杆方向 0 / 单位经济 -2

**禁止理由**：
- 双边市场需同时解决供需 liquidity，AI"自动撮合"不能解决信任、纠纷、质量审核
- 历史已证伪：OpenAI GPT Store 商业化失败、Google AI Studio Gallery 无人付费、Salesforce AgentExchange 无生态
- Marketplace 本征成功率 < 5%，叠加 AI hype 后更难

**典型陷阱叠加层**（识别红旗）：
- 用平台代币作为内部货币
- "睡后收入"、"AI 智能经纪自动结算"营销话术
- 同时讲 C 端技能交易 + B 端 API 市场 + Agent 接入层（缺乏聚焦）

**反面案例**：UUMit（同时踩中 4 条以上禁止条目）

**唯一豁免条件**：不做"双边 Agent 市场"，改做"单边 API 聚合"（RapidAPI / Apideck 模式，B 端调用方付费给单一聚合方）。

---

### 2.2 AI + Web3 / 代币化 AI 项目

**典型形态**：AI 算力代币、AI 能力代币、用代币结算的 AI 市场、链上 Agent 协议、"去中心化 AI 网络"。

**三维评分**：替代风险 -1 / 杠杆方向 -1 / 单位经济 -2

**禁止理由**：
- 监管风险极高（境内无法合规、海外面临 SEC 审查）
- 代币往往用来掩盖产品本身没有付费意愿
- 90% 以上"AI + Web3"项目的 AI 部分是装饰

**判断要点**：如果项目"必须用代币才能玩"，可 100% 判定 unit economics 不成立。

---

### 2.3 "AI 替代专业人士"类通用宣称

**典型形态**："AI 律师"、"AI 医生"、"AI 财务顾问"、"AI 心理咨询师"、"AI 设计师"等以"替代"为卖点的通用产品。

**三维评分**：替代风险 -1 / 杠杆方向 -1 / 单位经济 -2

**禁止理由**：
- 法律责任不可承担：错误的法律 / 医疗 / 税务建议产生的赔偿责任，早期创业公司无法承受
- 专业人士的真实价值是"责任承担 + 牌照 + 客户关系 + 行业人脉"，AI 无法替代
- 基础模型已具备 80% 通用专业知识能力，"做一个比 ChatGPT 更专业的律师 Bot" 是伪命题

**正确切法（豁免路径）**：
- 不做"AI 替代律师"，做"卖给律所的工作流工具"
- 或做"按结果交付的法律服务公司"（持牌经营，Outcome-as-a-Service）

---

### 2.4 AI 数字人 / 虚拟员工（通用版）

**典型形态**：AI 客服数字人、AI 直播主播、AI 虚拟员工等"3D 形象 + 模型 API"产品。

**三维评分**：替代风险 -2 / 杠杆方向 -1 / 单位经济 -1

**禁止理由**：
- HeyGen、Synthesia 已占据美国头部；中国市场被字节、阿里大厂官方数字人能力直接覆盖
- 模型语音 / 视频能力提升 = 用户直接用原生能力
- 技术栈高度同质化，最终演变为价格战

**唯一豁免条件**：在垂直场景拥有独家数据 / 牌照 / 客户关系（金融合规直播、医疗诊前问询、特定语种本地化客服）。

---

### 2.5 "AI 副业" / "睡后收入"类 C 端流量产品

**典型形态**：AI 做小说、AI 做短视频、AI 做电商、AI 带货等针对"想做副业的普通人"的流量产品。

**三维评分**：替代风险 -1 / 杠杆方向 -2 / 单位经济 -2

**禁止理由**：
- 目标用户 LTV 极低：副业人群付费意愿低、留存惨淡
- CAC 持续上涨：抖音 / 小红书副业流量获客成本 2024-2026 涨 3-5 倍
- 欺诈与监管风险：被分类为"灰产"概率高
- AI 能力贬值：人人都用 AI 做小说时，AI 做的小说就不值钱了

**残酷真相**：99% 的"AI 教你副业"项目，赚钱的是卖课的人，不是学的人。

---

## Tier 3 · 谨慎进入（单维负面，需特殊条件）

### 3.1 完全依赖单一模型公司 API 的产品

**触发条件**：产品 100% 调用 OpenAI / Anthropic / Google 其中一家的 API。

**风险**：价格风险（单方面调价）、能力风险（被原生覆盖）、政策风险（API 条款变更）

**最低标准**：
- 至少 2-3 个模型供应商的切换能力
- 预留 10-15% 开源模型部署比例
- Prompt / 工作流不硬编码到单一模型 API 特性

---

### 3.2 重度依赖 prompt 技巧的产品

**触发条件**：核心差异化是"我们的 prompt 写得好"或"独家 prompt engineering 技巧"。

**风险**：模型升级会让"prompt 技巧"价值快速归零（每 12 个月一轮）。

**唯一豁免条件**：prompt + 专有数据 + 深度工作流的组合护城河，而非单纯依赖 prompt。

---

### 3.3 中国团队远程做美国 B 端市场（无本地团队）

**触发条件**：团队在国内，目标客户是美国 SMB / Enterprise，但美国无本地销售 / CS / 法务团队。

**风险**：
- 美国 B 端销售必须本地化（电话、面对面、客户成功）
- 时区 + 文化 + 合规累积摩擦
- 中国背景在 SOC 2 / HIPAA / GDPR 上有信任劣势
- 地缘政治尾部风险

**例外**：纯 C 端、PLG 产品可以做（AI 生图、开发者工具、无账号 web 工具）。

---

### 3.4 以"研究 / 数据集 / 评估"为唯一卖点的开源项目商业化

**触发条件**：项目核心是开源数据集、benchmark、论文，希望以此商业化。

**风险**：研究价值 ≠ 商业价值。Stable Diffusion、LLaMA 都开源，但赚钱的不是发布者。

**豁免条件**：开源用作 GTM 而非核心 IP，商业化产品在开源基础上提供企业级运维 / 私有部署 / SLA / 训练数据闭环（HuggingFace、Weights & Biases 模式）。
