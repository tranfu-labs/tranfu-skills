# 典型案例库 · Case Library

> 真实项目的禁止清单对照分析，作为评估参照标本。

## 反面案例 · 踩雷标本

### Case 1：UUMit（中文 AI 能力交易市场）

**项目自我定位**：AI Native 能力网络 + Agent / MCP 接入层

**实际结构**：4 个业务捆绑（C 端技能交易 + 知识付费 + API 市场 + Agent 接入层）+ UT 代币

**踩雷对照**：

| 编号 | 禁止条目 | 命中 |
|------|---------|------|
| 1.2 | 通用 Agent / OS | ✅（A2A / MCP 定位） |
| 2.1 | AI 技能交易市场 | ✅（核心业务） |
| 2.2 | 代币化 AI | ✅（UT 代币） |
| 2.5 | "睡后收入"流量打法 | ✅（营销主线） |
| 聚焦原则 | 同时做 4 个业务 | ✅ |

**三维评分**：替代风险 -1 / 杠杆方向 -1 / 单位经济 -2 = **总分 -4 = 🔴**

**预后判断**：12-18 个月内退出市场或彻底转型为单一细分 B 端工具的概率 > 80%。

**教训提炼**：一家创业公司同时违反护城河、聚焦、单位经济、监管四个维度时，技术再好也救不回来。

### Case 2：Jasper.ai（AI 写作助手）

**项目自我定位**：企业级 AI 内容生成平台

**实际结构**：GPT-3 / GPT-4 的薄包装 + 模板库

**踩雷对照**：
- ✅ 1.1 单功能 AI 工具
- ✅ 3.1 完全依赖单一模型公司 API

**死亡轨迹**：
- 2022 年估值 15 亿美金（AI hype 顶峰）
- 2023 年 ChatGPT 直接覆盖核心功能
- 2024 年大规模裁员（70%+）、CEO 离职
- 2025 年彻底失去 AI 第一梯队地位

**教训提炼**：模板 + Prompt + 模型 API 的组合，不是产品，是过渡形态。

---

### Case 3：You.com（通用 AI 搜索）

**项目自我定位**：AI-first 搜索引擎

**踩雷对照**：
- ✅ 1.4 通用 AI 搜索

**死亡轨迹**：
- 2023 年估值高峰
- 2024 年 ChatGPT Search / Perplexity 双面挤压
- 2024 H2 裁员超 50%
- 2025 年转型 B 端 API（YouChat API）求生

**教训提炼**：通用搜索是模型公司核心战略，第三方做"中间层"无生路。

## 正面案例 · 三类幸存者

### Case 4：Cursor（第一类：品牌 + 切换成本）

**为什么活得好**：
- 开发者社区品牌（"Cursor 用户"成为身份标签）
- IDE 切换成本（一旦习惯 Cursor，回 VS Code 痛苦）
- 持续创新（Composer、Agent Mode）

**对照三类**：
- ✅ 第一类：基础模型公司碰不了的开发者品牌
- ✅ 第三类：每次 Claude / GPT 升级 Cursor 自动更强

**关键启示**：Cursor 本质上是 VS Code 的 fork + Anthropic API 的包装——但因为品牌和切换成本的护城河，模型公司很难直接吃掉。

---

### Case 5：Mercor（第二类：Outcome-as-a-Service）

**为什么活得好**：
- 不卖 AI 招聘工具，卖"招到的人"
- 按结果付费（per-hire）
- 承担质量责任（候选人离职兜底）

**对照三类**：
- ✅ 第二类：Anthropic 永远不会签"我帮你招到人"的合同

**关键启示**：把脏活累活做成 Outcome-as-a-Service，模型成本下降直接转化为毛利扩张。

---

### Case 6：LangSmith（第三类：杠杆共生）

**为什么活得好**：
- 给企业的 Agent / LLM 应用提供可观测性
- 模型 / Agent 越复杂，企业越需要 LangSmith
- 与所有模型公司中立（不绑定单一模型）

**对照三类**：
- ✅ 第三类：每次 Anthropic / OpenAI 升级，LangSmith 用户的需求增加

**关键启示**：基础设施位 + 中立性 + 模型不可知（model-agnostic）= 完美杠杆位。

## 评估练习模板

每次用禁止清单评估一个新项目时，建议先做对照练习：

1. **找最相似的反面案例**：这个项目和 UUMit / Jasper / You.com 哪个最像？
2. **找最相似的正面案例**：如果它要活下来，应该向 Cursor / Mercor / LangSmith 哪个学习？
3. **判定结构性距离**：当前结构离最相似的正面案例还差多少步？是可修复的还是基因不同？

如果"和反面案例的相似度 > 和正面案例的相似度"——立即停止启动。
