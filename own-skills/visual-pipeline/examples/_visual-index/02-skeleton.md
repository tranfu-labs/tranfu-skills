## 2. 状态机

每个状态明确各区域显示什么. 当前 HTML 实际渲染了**两个静态状态**, 加上桌面 wireframe 内可交互产生的若干派生状态. 列出最少三个能从 HTML 可达的状态:

- **入口态** (用户刚进入页面, 还没点任何卡片. HTML 桌面 wireframe "状态一", `selected = false`):
  - 头部摘要区 = 数据源路径 + runs 总数 + 崩溃总数 + 最近扫描时间 (内容铺满)
  - 主列表区 = 5 张 skill bar 卡片, 无任何卡片高亮; 头部含排序选择 (差值绝对值降序) + 崩溃过滤标签 (全部 / 只看崩溃)
  - 明细区 = **空状态**: "选择左侧任意一张 skill 卡片, 查看明细" (虚线灰盒占位)
  - 底部辅助区 = 占位 ("可选: 分页 / 上一 run 跳转")

- **选中态** (用户点了主列表里某张卡片, 例如 eval-news-impact. HTML 桌面 wireframe "状态二", `selected = true`):
  - 头部摘要区 = 同入口态 (顶部摘要不变)
  - 主列表区 = 同入口态, 但**选中行高亮** (蓝色描边 + 蓝色背景 + ring-2)
  - 明细区 = 填充: skill 头 + `62% → 78%`, Case 切换器 (Case 14 默认选中, 差值最大), Trace 模式切换器 (不启用 skill 默认), run.meta (286s · Token 100k · Pass), 时间戳翻页器 (默认落最新), Task / Chat / Result 三 tab (默认 Task, 二级 sub-tab 默认 `case-input`)
  - 底部辅助区 = 同入口态

- **崩溃过滤态** (用户在主列表区头部点 "只看崩溃" 过滤标签. HTML 中 `crash-filter` data-group + `aria-selected` 切换):
  - 头部摘要区 = 同入口态, 但"当前过滤状态摘要"字段反映"只看崩溃"
  - 主列表区 = 仅显示 `row.crash > 0` 的卡片 (HTML 里 `summarizer crash 1`, `broken-skill crash 2`; 其余隐藏)
  - 明细区 = 如已选中, 维持选中态; 否则维持入口态空状态
  - 底部辅助区 = 同入口态

- **tab 切换态** (选中态内部派生, Task / Chat / Result 三 tab 互斥):
  - 头部摘要区 / 主列表区 / 底部辅助区 = 同选中态
  - 明细区头部不变 (skill / Case / Trace / 时间戳), 明细区 tab body 切换至对应 tab 内容
  - HTML 中 `data-group="main-tab"` + 三个 `data-value` 互斥

- **时间戳翻页态** (选中态 / tab 切换态内部派生, 用户点 `‹` `›` 或下拉里某个历史时间戳):
  - 明细区时间戳行: 当前时间戳标签更新 (`[data-ts-current]` 全部刷成新时间戳), 下拉列表中对应时间戳被高亮
  - 明细区 tab body 内容应跟随时间戳变化 (HTML 内 mock 数据未真实跟随, 但信息架构上是同 case+模式下相邻时间戳切换)

## 3. 区域间关系

(渲染顺序 / 选中联动 / 数据依赖. 来源: HTML `flows` 数组 7 条 + 实际 DOM 行为.)

- **主列表区选中卡片 → 明细区内容刷新** (Flow "钻入明细")
- **主列表区过滤标签 → 主列表内列表过滤** (Flow "Filter Failed"); 不动明细区 (如已选中则维持)
- **主列表区排序选择 → 主列表内顺序重排**, 不动其它区域
- **明细区内 Case 标签切换 → 明细区 Trace 行元信息刷新 + 明细区 tab body 内容刷新** (Flow "切换 case")
- **明细区内 Trace 模式切换 → 时间戳回到最新 + 明细区 tab body 内容跟随模式变化** (Flow "切换 trace 模式")
- **明细区内时间戳翻页 → 当前时间戳切换 + 明细区 tab body 元信息和内容刷新** (Flow "翻时间戳")
- **明细区内三 tab 互斥**: 切 tab **保留** skill / case / 模式 / 时间戳 上下文 (Flow "切换 Tab")

**区域间不联动** (明确写出, 避免后续脑补): 头部摘要区不被任何主列表区/明细区内动作改变; 底部辅助区当前完全独立 (因为只是占位).

## 4. 浏览路线落到区域

(把「显示信息段」第 3 节的动线落到具体区域)

- **扫读路线** = 进入 (入口态) → 头部摘要区扫顶部数字 → 主列表区按差值绝对值排序扫双 bar 和大数字 → 选定 skill 点入 → 选中态
- **调试路线** = 选中态 (主列表区高亮 + 明细区默认 Task) → 明细区内 Case 标签选差值最大 case → 明细区内 Trace 选模式 → 明细区切 Chat (展开 ✗ 段执行日志) → 明细区切 Result (最终文档 / 结构化产物 / 判定)
- **崩溃排查路线** = 入口态 → 主列表区头部过滤标签点 "只看崩溃" (崩溃过滤态) → 主列表区仅余崩溃卡片 → 点卡片 (选中态+崩溃过滤态叠加) → 明细区同调试路线
- **产物阅读路线** = 选中态 → 明细区三 tab 横扫: Task (case-input + 三段 instruction sub-tab) → Chat (system/user/assistant + 折叠日志) → Result (三个 sub-section: 最终文档 / 结构化产物 / 判定)
- **跨时间戳比较退化版** = 选中态 → 明细区时间戳翻页器 `‹` / `›` 或下拉, 当前时间戳切换 → 明细区 tab body 内容跟随 (Flow "翻时间戳")

## HTML 输出要求 (`02-skeleton.html`)

- **顶部明确声明**: "本页所有内容为模拟数据, 仅用于展示信息密度与排版"
- **灰盒**: 仅中性灰阶 (#fff / #f5f5f5 / #ccc / #888 / #444), 不上品牌色 (高亮行可用淡蓝)
- **每个区域** 标中文区域名
- **中间用真实模拟数据** (中文 + 真实结构, 不是 lorem, 不是 `[字段名]` 占位符)
- **旁边图例** 解释每条模拟数据对应哪个字段 (字段名来自「显示信息段」第 4 节)
- **多状态** 上下排列 (不用 tab 切换, 此阶段保持静态对照)
- 每个状态前用 H2 标题: "入口态 / 选中态 / 崩溃过滤态" 等
