# `_visual-index` — visual-pipeline 反向示例

这是 `web-goal-docs/_visual/index.html` (UI 审查协议 r2 - Layer 查看器) 的**反向**拆解, 用来给后续真正跑这条 skill 的人**看每段产物长什么样**.

**不是教程**, 是**参照**: 看到产物去对照, 而不是照抄.

**全程中文**, 不用英文缩写.

## 反向意味着什么

正向流程: 第一段 → 第二段 → 第三段 → 选定版 HTML.

反向 = 已有选定版 HTML, 倒推前三段文档应当长什么样. 因此本目录的特殊处理:

- `03-style.md` **只描述当前选定版的调性**, 不像正向流程那样造两版候选 (反向已经没有"挑"的余地)
- `03-selected.html` 是 `_visual/index.html` 的**原样拷贝**, 不修改一字
- 上游 `02-skeleton.html` 是按选定版反推出的灰盒, 用来对照"如果只看灰盒 wireframe, 它长什么样"

## 文件清单

| 文件 | 内容 |
|---|---|
| `README.md` | 本文件 |
| `01-display-info.md` | 反推用户画像 / 场景 / 浏览路线 / 字段清单+优先级 |

| `02-skeleton.html` | 灰盒 wireframe (按反推的区域 + 状态机绘制, 中间是真实中文模拟数据, 旁边图例标对应字段) |
| `03-style.md` | 性质判定 + 当前选定版的调性提取 (色板 / 字号 / 留白) |
| `03-selected.html` | 原样拷贝自 `_visual/index.html` (1061 行) |

## 看完之后该带走什么

1. 字段清单**长什么样** — 真字段名 + 真数据源, 不是 lorem
2. 状态机**写多细** — 入口态 / 选中态 / 崩溃过滤态最少三个, 每个区域在每个状态下显示什么必须写明
3. 灰盒 wireframe **不上色**, 中间放真实中文模拟数据, 旁边图例标"这条数据对应字段 X", 状态上下排列对照
4. 风格 token **可追溯** — 文档里写的色值 / 字号必须能在选定版 HTML 的 CSS / 类名里 grep 到
