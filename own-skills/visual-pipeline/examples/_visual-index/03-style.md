# 三、风格 + 实践 — UI 审查协议查看器

> **目的**: 判定页面性质 → 列组件清单 → 反推该页面的调性 → 标出选定文件.
> **反向特殊**: 此 demo 不造两版候选 (页面已经定型, 没有挑的余地), 只描述选定版当下的调性. 但**新增「组件清单」一节**, 体现「风格 + 实践」段应有的完整流程: 组件→样例→整体组合.
> **配套**:
> - `03-selected.html` = `_visual/index.html` 原样拷贝 (持续迭代至像素级的产物)
> - `03-alt-设计感衬线.html` = 按新模板补出的参考形态, 含「组件样例库」+「整体组合」两 section. 风格对标 `web-goal-docs/design-ref/image.png`.

**全程中文**, 不用英文缩写.

## 1. 页面性质判定

- **位置**: **混合 (实用 + 设计感)** — 严格按场景反推应偏实用 / 数据密度, 但用户给出的参考图 `web-goal-docs/design-ref/image.png` 是"衬线大字 + 大留白 + 单点鲜蓝"的设计感作品集调性. demo 反映用户决策, 因此朝设计感 + 留白方向走.
- **理由**: 主场景「批后扫读」「调试单 run」是工程师扫读 + 钻取 (引用 `01-display-info.md` 第 2 节). 单纯按这个推, 应该是 GitHub Actions 那种紧密灰阶仪表盘. 但用户希望对标 `web-goal-docs/design-ref/image.png`, 取舍的是"信息密度让位给阅读舒适度和设计调性". 本 demo 体现这层取舍, 不当判官.

## 2. 组件清单 ⭐

## 3. 调性候选

(反向 demo 不造两版, 只描述用户决定对标 `web-goal-docs/design-ref/image.png` 之后的唯一调性.)

### 候选「设计感 · 衬线留白」(唯一)

- **对标**: `web-goal-docs/design-ref/image.png` — TAMAKO LAB 作品集页面. 抽出的关键风格点:
  1. 大字号黑色**衬线 (serif)** 标题 (Icon Design ~80px), 字距紧凑, 细衬线脚, 视觉重量大
  2. 副标小衬线 italic (Design), 跟在大标题肩上, 像作者签名
  3. 顶部导航全大写无衬线 small caps + 宽字距, 居右
  4. **米灰暖底** + 纯白卡片浮起, 卡片**大圆角 + 柔和大阴影**, 卡片间距极松
  5. 单一**鲜亮蓝**作为强调色 (按钮 / 主图标), 其余皆为黑白灰
  6. 黑色长胶囊底栏装署名 + 年份, 圆角 999
  7. **留白极松**: 卡片四周大量负空间, 内 padding 也大, 信息密度低但呼吸感强
- **不对标**: GitHub Actions / Sentry / Vercel 一类工程仪表盘 (虽然功能场景更贴, 但用户决策朝设计感方向)
- **色板** (实测自 `design-ref/image.png`):
  - 页面底色: `#ECEAE5` (米灰暖底)
  - 卡片底色: `#FFFFFF`
  - 主文字: `#0A0A0A` (近黑)
  - 次文字: `#7A7A7A` (中灰)
  - 弱文字 / 边线: `#C9C5BE` (米灰边)
  - 强调蓝: `#3B6BFF` (按钮 + 主图标)
  - 浅蓝选中底: `#E8EEFF`
  - 暗组 (底栏胶囊): `#0A0A0A`
  - 失败红: `#B0341E` 字 / `#FBEDE8` 底
  - 成功绿: `#1F6F4A` 字 / `#E7F2EC` 底
- **字号比例 (像素)**:
  - 衬线大标题: 64-72 (Section / 主区域大标题)
  - 衬线副标题 italic: 22-24
  - 卡片标题 (serif): 22-28
  - 正文: 14-15
  - 次文字 / 元信息: 12
  - 全大写导航 / 小标签: 11 (字距 0.1em)
- **留白密度**: **松**. Section 间 80px, 卡片间 32px, 卡片内 padding 28-32px
- **控件特征**: 卡片**大圆角** 20-24px, 按钮圆角 10px, 胶囊圆角 999; 卡片有**柔阴影** `0 12px 32px rgba(20,20,20,.08)`; 边线只用浅米灰; 衬线用 `Noto Serif SC` (Google Fonts) → fallback `Source Han Serif SC` / `Georgia` / `serif`
- **HTML**: `03-alt-设计感衬线.html` (内含「组件样例库」+「整体组合 - 选中态」两 section)

## 4. 选定 + 原因 ⭐

- **选定**: 候选「设计感 · 衬线留白」
- **理由**: 用户原话 "你的风格要参照它" 指向 `web-goal-docs/design-ref/image.png`, 即把审查器朝衬线大字 + 米灰底 + 鲜蓝强调 + 大留白方向重做. 严格说反场景里"不做主视觉聚焦物 / 不做营销区块 / 不做玻璃面板"会和这个方向冲突, 但 demo 反映的是用户的决策, 不替用户做平衡.
- **选定文件**: `03-selected.html` = `_visual/index.html` 原样拷贝 (1061 行, 仍为工程仪表盘形态, 见末尾 demo 限制说明)

## demo 限制说明

本反向 demo 中, `03-selected.html` 沿用既有产物 `_visual/index.html` 的整体形态 (工程仪表盘紧凑灰阶), 是反向 demo 的**固定约束**, **不会**真按本 alt 的风格重做. `03-alt-设计感衬线.html` 体现的是"如果按参考图 `web-goal-docs/design-ref/image.png` 重做这套审查器, 它应该长什么样" — 即顶部一节「组件样例库」逐个展示每个组件类型 (套衬线大字 + 米灰底 + 鲜蓝强调调性), 底部一节「整体组合 - 选中态」用组件 + 真字段 + 头部摘要区 / 主列表区 / 明细区拼起来.
