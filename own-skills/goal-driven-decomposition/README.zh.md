# 目标驱动分解

把「做一个 X」这类模糊目标, 先押注最可能的形态, 再让画像 Agent 反复核对, 一路推到真产物。

## 什么时候用它

**冷启动首次**:

我丢一句「做一个 X」或「build a tool」, 项目里还没有 `goal-docs/`, 想让它先押注最可能的形态、再提议一层更深的真实目标, 我点头再往下走。

**咨询开场**:

我在问「你会怎么做 X」「先讨论一下」, 还没决定要不要做, 想让它顺势展开画像和维度, 让我看清楚才拍板。

**迭代修正**:

我说「X 偏了」「首页那块不对」, `goal-docs/` 已经在, 想让它跳过重新设计, 把这次的偏差直接写进验证 - 修复循环。

**迭代新增**:

我说「加个 Y」「还要支持 Z」, 想让它只在受影响的维度上重跑, 不要把其他维度也一并翻新。

**画像变更**:

我说「这次面向的用户换成 Z」, 之前的画像基本作废, 想让它停下来问我: 全部重跑、只选几个维度、还是另开一份新的 `goal-docs/`。

**不接**:

零设计维度的一次性小改 (改 typo / 换变量名) → 不触发; `goal-docs/` 已在但我只是问问、不动 → 不触发; 商业该不该做 → **business-analysis-pipeline**; 审文章草稿像不像营销号 → **credibility-review**; 把 skill 发到公司库 → **tranfu-publish**。

## 它会产出什么 / 你会看到什么

**默认先出方案, 想改写目标一定要经过用户点头**——最反常识的一点。用户原话原样存在 `user_goal_surface.md` 里当审计镜, 一层更深的真实目标只有拿到用户同意才落到 `00-true-goal-r{N}.md`。

- **落盘 `goal-docs/`**: `user_goal_surface.md` 存原话, `00-true-goal-r{N}.md` 存真实目标, `00-用户画像-r{N}.md` 存可扮演画像, `20-goal-tree-r{N}.md` 存目标树与切片, 按维度落 `0X-{维度}-r{N}-{对话轮}-{角色}.md`, 收尾还有 `40-build-plan`、`90-validation-plan`、`91-validation-run`、`92-fix-attempt`、`99-retro`
- **本地截图入库**: 走样张的维度 (UI / 视觉 / 排版 / 文案基调) 必须在 `goal-docs/_refs/` 下先放 1-3 张正例截图 + 1-3 张反例截图, 只给一个产品名不算
- **PNG 候选样张**: 走样张的维度每一维都要先渲染 PNG 让画像 Agent 反应, 光改文字方案不算数
- **多个新开的 Agent**: 画像、范围、DoD、目标树、构建方案、验证、修复各拿一个新上下文, 主 agent 不亲手写产物内容
- **真产物**: 最后是真的代码 / 站点 / CLI, 不是假图, 不是截图
- **绝不会做**: 没问过用户就悄悄改写原话; 动 `git` (不 commit / push / 切分支); 发任何外部通知; 每维度的验证 - 修复循环超过 3 轮还没过 → 主动浮到用户, 绝不静默继续

## 前置条件 / 边界

**前置**:

一个可写目录 (会新建 `goal-docs/`); 涉及样张维度 (UI / 视觉 / 排版 / 文案基调) 时, 需要先在 `goal-docs/_refs/` 下放 1-3 张正例截图 + 1-3 张反例截图。

**相邻 skill 分工**:

| 动作 | 交给 |
|---|---|
| 冷启动搭 `AGENTS.md` / `openspec/` 骨架 | **project-init-docs** |
| 已有 `openspec/` 项目里跑单次 change 闭环 | **openspec-driven-development** |
| AI 产品该不该做 / 商业可行性 | **business-analysis-pipeline** |
| 审一篇文章草稿像不像营销号 | **credibility-review** |
| 把本地 skill 发到公司库 | **tranfu-publish** |

**不接的场景**:

- 零设计维度的一次性小改 (改 typo / 换变量名)
- `goal-docs/` 已在, 但我只是问问、不改也不加
- 已经在别的 skill 作用域 (商业评估 / 审稿 / 发布)

**微妙边界**:

- 冷启动 vs 迭代: 看 `goal-docs/` 里有没有 `00-用户画像-r{N}.md`, 缺就是冷启动
- 枚举型 vs 样张型 维度: 规格能不能无损写成文字, 能就走枚举 (功能 / 技术 / 部署); 不能就走样张 (UI / 视觉 / 排版 / 文案基调) → 强制渲染 PNG + 盲选比对
- 「加个 X」若目标树里根本没有对应模块 → 分配下一个空号并追加, 不会挤占预留段
