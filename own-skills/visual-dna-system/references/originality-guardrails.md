# Originality Guardrails

Use this reference to decide what may transfer from a visual sample and what must not transfer.

## Transferable

Transfer abstract design principles:

- Mood, energy, density, and trust posture.
- Color role behavior such as warm-neutral backgrounds, restrained accents, or high-contrast states.
- Typography personality and hierarchy rhythm without copying branded fonts when they are distinctive.
- Spacing rhythm, grid logic, alignment tendencies, and scan path.
- Abstract component language such as compact controls, editorial panels, dense data rows, or soft cards.
- Material qualities such as low shadows, thin borders, paper-like surfaces, or glass-like layering when not source-identifying.
- Imagery logic such as documentary photography, editorial crops, abstract placeholders, or diagram style.
- Motion feel such as calm fades, snappy transitions, or restrained hover feedback.
- Composition grammar such as asymmetry, generous negative space, modular repetition, or cinematic sequencing.

## Non-Transferable

MUST NOT transfer:

- logos, wordmarks, mascots, slogans, or brand names.
- exact layouts that are distinctive to the source.
- proprietary components, navigation, command structures, or signature UI patterns.
- exact copy, product names, data, screenshots, or marketing claims.
- Source-specific icon sets, illustrations, photos, or brand assets.
- Protected visual identity markers that make the output look like the source owner.

## Rewrite Rule

When a source detail is attractive, rewrite it as a principle:

```text
source detail -> abstract principle -> original implementation
```

Example:

```text
same hero with same logo and same layout -> calm premium hero, soft hierarchy, restrained copy density -> new composition with different structure, content, identity, and assets
```

## Self-Check

Before delivery, ask:

- Could a viewer mistake the output for the source brand?
- Did any source-specific layout, asset, copy, or component survive unchanged?
- Does the downstream producer still need to see the original sample to use this system?
- Are the non-transferable items explicit enough to prevent accidental copying?

If any answer is not a clear "No", revise the transferable/non-transferable split before delivery.
