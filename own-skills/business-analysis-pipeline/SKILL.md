# business-analysis-pipeline

## When to use

当用户想对一个 **AI 产品概念 / 创业方向 / 投资标的**做系统化商业评估时触发. 典型触发短语:

- "评估这个 AI 产品的商业可行性: <描述>"
- "帮我跑一份商业分析报告"
- "出一份 AI 创业方向评估"
- "投资尽调一下这个项目"
- "我有几个产品想法, 横向对比一下哪个值得做"

不适用: 已上线产品的运营复盘 / 单纯做市场调研 / 只要财务建模 — 本 skill 是**前期方向评估**的方法论 pipeline, 不是落地 SOP 也不是财务模型.

- [credibility-review](../../own-skills/credibility-review/SKILL.md) — 用双轨 agent 审"踩坑记/养成记"文章可发性；**本 skill 区别**：作用对象是产品/项目概念, 不是文章

- 每模块限 2-3 个最关键要点, 砍掉常见 PEST/SWOT 的清单堆砌

## Steps (核心工作流)

### Step 1：PEST 宏观环境分析

**角色：** 专业商业分析师
**任务：** 对用户提供的 AI 产品进行 PEST 宏观环境分析

**四个维度：**
- P（政治与监管）：AI 监管政策、行业准入、政策支持、国际合规
- E（经济）：市场规模 TAM/SAM/SOM、客户支付意愿、AI 成本趋势、融资环境
- S（社会与文化）：用户 AI 认知与接受度、隐私态度、工作方式变迁
- T（技术）：基础模型能力、技术成熟度、开源 vs 闭源、基础设施成熟度

每个维度评"有利度"（1-5 分），只聚焦与该产品最相关的 2-3 个要点。

**输出格式：**
1. PEST 评分总览表（维度 | 评分 | 核心判断）
2. 每个维度简要分析（每维度 3-5 句话，聚焦对该产品的具体影响）
3. 3 个关键发现，编号 P-KF-1 到 P-KF-3
4. 一句话综合结论

使用 🟢🟡🔴 标记。全部中文。
严格限制：只输出分析内容，不要提议后续步骤或额外产出物。

### Step 2：波特五力竞争分析

**角色：** 专业商业分析师
**任务：** 对用户提供的 AI 产品进行波特五力竞争分析
**依赖：** 引用已完成的 PEST 关键发现（P-KF 编号）来支撑分析

**五力维度（各评 1-10 分威胁度）：**
1. 现有竞争者强度：直接竞品、大厂布局、差异化程度
2. 新进入者威胁：数据壁垒、技术壁垒、资金壁垒
3. 替代品威胁：传统方案、通用 AI 助手、开源方案
4. 买方议价能力：客户集中度、切换成本、价格敏感度
5. 供应商议价能力：模型供应商集中度、算力资源、人才稀缺度

每个力量只分析最关键的 1-2 个因素。

**输出格式：**
1. 五力评分总览表（力量 | 威胁评分 | 核心判断 | 应对思路）
2. 主要竞品对比表（3-5 个竞品，产品 | 差异化 | 威胁等级）
3. 3 个关键发现，编号 F-KF-1 到 F-KF-3
4. 竞争格局一句话结论

使用 🟢🟡🔴 标记。全部中文。
严格限制：只输出分析内容，不要提议后续步骤或额外产出物。

### Step 3：SWOT 综合评估

**角色：** 专业商业分析师
**任务：** 对用户提供的 AI 产品进行 SWOT 综合分析
**依赖：** O（机会）和 T（威胁）从 P-KF 和 F-KF 中提取，标注来源编号

**规则：**
- S（优势）和 W（劣势）基于团队/产品内部能力评估
- 各象限评分 1-5 分，每象限列出最重要的 2-3 条

**战略矩阵（每类给出 1 条最关键策略）：**
- SO 战略（进攻型）、ST 战略（防御型）、WO 战略（扭转型）、WT 战略（保守型）

**输出格式：**
1. SWOT 四象限表（象限 | 评分 | 要点列表）
2. 战略矩阵表（策略类型 | 策略内容 | 优先级）
3. 3 个关键发现，编号 S-KF-1 到 S-KF-3
4. 推荐战略方向一句话结论

使用 🟢🟡🔴 标记。全部中文。
严格限制：只输出分析内容，不要提议后续步骤或额外产出物。

### Step 4：商业模式画布（BMC）

**角色：** 专业商业分析师
**任务：** 对用户提供的 AI 产品进行商业模式画布分析
**依赖：** 基于前序 SWOT 战略方向来设计商业模式

**九大模块（每模块用 1-2 句话概括核心要点）：**
1. 客户细分（CS）
2. 价值主张（VP）
3. 渠道通路（CH）
4. 客户关系（CR）
5. 收入来源（RS）
6. 核心资源（KR）
7. 关键业务（KA）
8. 重要合作（KP）
9. 成本结构（CS）

重点展开：价值主张、收入来源、成本结构。其余模块简要概括。

**输出格式：**
1. 商业模式画布总览表（9 模块，每模块核心要点）
2. 价值主张、收入来源、成本结构各展开 3-5 句详细分析
3. 单位经济模型关键指标（CAC、LTV、LTV/CAC、毛利率估算）
4. 3 个关键发现，编号 B-KF-1 到 B-KF-3
5. 商业逻辑是否自洽的一句话判断

使用 🟢🟡🔴 标记。全部中文。
严格限制：只输出分析内容，不要提议后续步骤或额外产出物。

### Step 5：机会评估矩阵

**角色：** 专业商业分析师
**任务：** 对用户提供的 AI 产品进行机会评估矩阵打分
**依赖：** 基于所有前序分析结果（P-KF、F-KF、S-KF、B-KF）进行综合评分

**评分维度（5 大维度，总分 120 分）：**

| 维度 | 满分 | 评分项 | 评分标准 |
|------|------|--------|---------|
| 市场维度 | 30 | 市场规模 TAM（10）| 1-2:<1亿, 3-4:1-10亿, 5-6:10-50亿, 7-8:50-100亿, 9-10:>100亿 |
| | | 市场增速（10）| 1-2:负增长, 3-4:<10%, 5-6:10-20%, 7-8:20-50%, 9-10:>50% |
| | | 市场时机（10）| 1-3:过早/过晚, 4-6:尚可, 7-8:良好, 9-10:完美窗口 |
| 竞争维度 | 20 | 竞争格局（10，反向）| 1-3:红海, 4-6:有竞争未固化, 7-8:蓝海, 9-10:无直接竞争 |
| | | 差异化壁垒（10）| 1-3:难建立, 4-6:壁垒薄, 7-8:可建立, 9-10:独特优势 |
| 产品维度 | 25 | 技术可行性（10）、产品价值清晰度（10）、AI 独特价值（5）| |
| 商业维度 | 25 | 商业模式清晰度（10）、盈利潜力（10）、单位经济健康度（5）| |
| 执行维度 | 20 | 团队匹配度（10）、MVP 可实现性（10）| |

**评级标准：** 96-120=S 级（强烈推荐），78-95=A 级（值得推进），60-77=B 级（谨慎推进），42-59=C 级（风险较高），<42=D 级（建议放弃）

**输出格式：**
1. 综合评分表（维度 | 得分 | 满分 | 引用关键发现 | 一句话依据）
2. 综合评分与评级
3. 一段综合评估结论（3-5 句话）

使用 🟢🟡🔴 标记。全部中文。
严格限制：只输出分析内容，不要提议后续步骤或额外产出物。

### Step 6：战略建议

**角色：** 专业商业战略顾问
**任务：** 基于所有前序分析结果，为 AI 产品提供战略建议
**依赖：** 引用各模块关键发现编号（P-KF、F-KF、S-KF、B-KF）支撑建议

**只需覆盖以下 4 个方面，每方面 3-5 句话：**
1. 核心战略定位：推荐方向 + 差异化定位
2. 产品策略：MVP 核心功能 + 产品路线图（V1→V2→V3 各一句话）
3. 市场进入策略：GTM 策略 + 定价建议
4. 关键风险与应对：列出 Top 3 风险及应对措施（表格）

**输出格式：**
1. 各方面简要分析
2. 风险表（风险 | 概率 | 应对措施）
3. 近期行动计划（0-3 个月，3-5 条具体任务）

全部中文。
严格限制：只输出分析内容，不要提议后续步骤或额外产出物。

### Step 7：执行摘要

**角色：** 专业商业分析报告撰写专家
**任务：** 基于所有前序分析结果撰写执行摘要

**执行摘要内容（控制在 400-600 字）：**
1. 标题行：# [产品名称] 商业分析报告
2. 综合评分与评级（从机会评估矩阵提取）
3. 核心发现（3-5 句话）：最大优势、最大机会、最关键风险、商业逻辑是否成立
4. 关键数据指标表（维度 | 评分 | 判断）
5. 一句话决策建议（明确、不模棱两可）

使用 🟢🟡🔴 标记。全部中文。
严格限制：只输出执行摘要，不重复详细分析内容。

## 关键发现编号体系

各模块输出的关键发现采用统一编号, 供下游模块交叉引用, 确保分析逻辑链条连贯:

| 模块 | 编号格式 | 示例 |
|------|---------|------|
| PEST | P-KF-1 ~ P-KF-N | P-KF-1：国内 AI 监管趋严但政策支持力度大 |
| 波特五力 | F-KF-1 ~ F-KF-N | F-KF-1：大厂已布局但垂直领域尚未固化 |
| SWOT | S-KF-1 ~ S-KF-N | S-KF-1：技术优势突出但品牌认知不足 |
| BMC | B-KF-1 ~ B-KF-N | B-KF-1：订阅模式 LTV/CAC > 3 商业逻辑自洽 |

**引用规则:**
- SWOT 的 O/T 必须引用 P-KF 或 F-KF
- 战略建议必须引用各模块 KF 编号
- 机会评估矩阵的每个维度得分必须标注依据的 KF 编号

## 输入要求

- **必填**: 产品名称 / 产品概念描述 / 核心功能
- **选填**: 目标用户 (个人消费者 / 中小企业 / 大型企业 / 特定行业从业者) / 商业模式 (订阅 / 按量 / 免费增值 / 企业授权) / 已知竞品 / 团队优势 / 目标市场 (中国 / 全球 / 特定地区) / 分析深度 (快速 / 标准 / 深度) / 模块开关

## 质量控制规范

1. 全文使用中文
2. 每个分析框架结论都包含产品名称和具体场景
3. 市场数据标注来源 (公开报告 / 估算 / 推断)
4. 使用表格呈现结构化数据
5. 使用 🟢 (利好/高分) 🟡 (中性/中等) 🔴 (风险/低分) 标记重要指标
6. 建议包含具体行动而非原则性描述
7. 各模块控制输出长度, 避免冗余重复

## 使用建议

- 产品描述越具体, 分析质量越高
- 提供已知竞品信息可显著提升竞争分析准确度
- 说明团队优势有助于 SWOT 评估和团队匹配度评分
- 多个产品概念可以一起提交进行横向对比
- 分析深度可选: 快速 (重点结论) / 标准 (完整分析) / 深度 (详尽展开)

## What NOT to do

- ❌ 跳过模块直接评分 — 综合评分必须有前序 KF 支撑
- ❌ 编造市场数据 — 来源不明的必须标注 "估算" / "推断"
- ❌ 各模块输出"后续步骤建议" — 严格限定只输出本步分析
- ❌ 评级模棱两可 (eg "可能值得做") — 必须给 S/A/B/C/D 五级中的一档
- ❌ 重复堆砌 PEST/SWOT 通用清单 — 每维度只挑 2-3 个最相关要点
- ❌ 输出英文术语而不解释中文含义
- ❌ 对已上线产品做运营复盘 (本 skill 是前期评估, 不是事后分析)
